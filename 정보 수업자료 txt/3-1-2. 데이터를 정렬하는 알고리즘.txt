탐색 알고리즘
탐색(search)이란 방대한 데이터에서 주어진 조건에 맞는 데이터를 찾는 과정이다. 탐색 알고리즘은 데이터를 탐색하는 논리적인 순서를 의미하며, 대표적인 탐색 알고리즘에는 순차 탐색과 이진 탐색 등이 있다.
정렬된 자료: 1, 4, 5, 7, 9, 13, 16

① 순차 탐색
순차 탐색(sequential search)은 데이터를 앞에서부터 차례대로 비교하여 원하는 데이터를 찾는 탐색 알고리즘이다. 도서 번호가 오름차순으로 정렬된 상태에서 '9번 도서'를 순차 탐색으로 찾는 방법은 다음과 같다.

첫 번째 데이터(1)와 탐색할 데이터(9)를 비교한다. 찾는 데이터와 일치하지 않으므로 다음 데이터로 이동한다.

두 번째 데이터(4)와 탐색할 데이터(9)를 비교한다. 찾는 데이터와 일치하지 않으므로 다음 데이터로 이동한다.

세 번째 데이터(5)와 탐색할 데이터(9)를 비교한다. 찾는 데이터와 일치하지 않으므로 다음 데이터로 이동한다.

네 번째 데이터(7)와 탐색할 데이터(9)를 비교한다. 찾는 데이터와 일치하지 않으므로 다음 데이터로 이동한다.

다섯 번째 데이터(9)와 탐색할 데이터(9)를 비교한다. 찾는 데이터와 일치하므로 탐색을 종료한다. (비교 5회)

순차 탐색은 비교적 단순한 알고리즘으로, 이해하기 쉽고 정렬되지 않은 데이터에도 적용할 수 있다. 하지만 데이터가 배열된 상태에 따라 비교 횟수가 달라진다. 찾고자 하는 자료가 앞쪽에 위치한다면 적은 비교 횟수만으로도 원하는 자료를 빠르게 탐색할 수 있지만, 자료가 뒤쪽에 위치할수록 비교 횟수는 많아진다.

[추가 정보]
Q: 데이터가 1억 개일 때 순차 탐색으로 원하는 데이터를 몇 번 만에 찾을 수 있을까?
A: 원하는 데이터가 가장 앞에 있는 경우 1회, 원하는 데이터가 마지막에 있는 경우 1억 회를 비교하므로, 평균 탐색 횟수는 약 5,000만 번이 된다.

이번 페이지는 **'이진 탐색'**에 관한 내용이네요. 마찬가지로 한 번에 복사해서 사용하시기 좋게 텍스트로 정리해 드립니다.

프로그램코드:
#순차 탐색

def linear_search(arr, target):
    for i in range(len(arr)):
        print(f'실행횟수: {i+1}')
        if arr[i] == target:
            return i  # 인덱스 반환
    return -1  # 못 찾으면 -1

data = [3, 7, 1, 9, 5]

print(f'실행 결과 타겟값의 위치: {data} 리스트 인덱스 {linear_search(data, 9)}에 존재함.')

② 이진 탐색
이진 탐색(binary search)은 데이터가 정렬된 상태에서 범위를 반씩 좁혀가며 원하는 데이터를 찾는 탐색 알고리즘이다. 도서 번호가 오름차순으로 정렬된 상태에서 '9번 도서'를 이진 탐색으로 찾는 방법은 다음과 같다.

1. 전체 데이터 중 가운데 위치한 데이터(7)와 탐색할 데이터(9)를 비교한다. 데이터(7)보다 데이터(9)가 크므로 데이터(7)의 오른쪽으로 이동한다.
2. 데이터(7)의 오른쪽에 있는 모든 데이터 중 가운데 위치한 데이터(13)와 탐색할 데이터(9)를 비교한다. 데이터(13)보다 데이터(9)가 작으므로 데이터(13)의 왼쪽으로 이동한다.
3. 마지막으로 좁혀진 탐색 범위 데이터(9)와 탐색할 데이터(9)를 비교한다. 일치하므로 탐색을 종료한다. (비교 3회)

이진 탐색은 전체 탐색 범위의 중간 위치를 기준으로 두 부분으로 분할하고, 찾고자 하는 데이터와 중간 위치의 데이터를 비교하여 다음 탐색 범위를 결정하는 방식이다. 순차 탐색을 이용해 '9번 도서'를 찾을 때 비교 횟수는 5회이지만 이진 탐색을 이용하면 3회로 줄어 알고리즘 효율성이 높아진다. 이진 탐색은 찾고자 하는 데이터가 어떤 위치에 있더라도 탐색 효율성이 좋다는 장점이 있지만, 데이터를 탐색하기 전에 미리 데이터가 정렬되어 있어야 한다는 전제 조건이 필요하다.

[데이터 개수가 짝수인 경우]
데이터의 개수가 짝수일 경우 중간값이 명확하게 결정되지 않기 때문에 중간값을 결정하는 방식을 미리 설정하고 일관되게 해당 규칙을 적용하는 것이 중요하다. 중간값의 왼쪽 데이터를 기준으로 할지, 오른쪽 데이터를 기준으로 할지 결정할 수 있다. 예를 들어 데이터가 8개일 때 중간값의 왼쪽 데이터를 기준으로 결정하면 4번째 데이터가 기준이 되고, 오른쪽 데이터를 기준으로 하면 5번째 데이터가 기준이 된다.

[추가 정보 및 주의 사항]
- 주의: 이진 탐색으로 13을 탐색하는 경우는 2회 만에 찾을 수 있다. 동일한 알고리즘이라도 찾고자 하는 데이터가 무엇이냐에 따라 탐색 횟수가 달라진다.
- Q: 데이터가 1억 개일 때 이진 탐색으로 원하는 값을 최대 몇 회 만에 찾을 수 있을까?
- A: $2^n = 1$억을 만족하는 값을 계산하면, $2^n = 100,000,000$, $n = \log_2(100,000,000)$. $n$이 약 26.57로 최대 27번의 이진 탐색을 수행하면 된다.
- 말풍선 내용: "데이터의 개수가 짝수일 경우 중간값을 뭘로 하지?" / "왼쪽을 기준으로 하면 8번 도서가 중간값이 되고, 오른쪽을 기준으로 하면 10번 도서가 중간값이 되는 거야."

프로그램 코드:
# 이진 탐색
def binary_search(arr, target):
    left = 0
    right = len(arr) - 1
    count = 0

    while left <= right:
        mid = (left + right) // 2
        count += 1
        print(f"{count}번째 시도: mid={mid}, 값={arr[mid]}")

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1


data = [1, 3, 5, 7, 9, 11]
print("결과 인덱스:", binary_search(data, 7))

2. 데이터에 따라 달라지는 탐색 알고리즘 효율성
주어진 데이터가 어떤 기준으로 정렬되어 있는지와 탐색하고자 하는 데이터가 무엇인지에 따라 탐색 알고리즘의 효율성이 달라진다. 다음 두 경우에서 4를 탐색할 때, 탐색 알고리즘별로 비교 횟수를 살펴보자.

[경우 1] 1, 4, 5, 7, 9, 13, 16 (오름차순)
- 순차 탐색: 비교 횟수 2
- 이진 탐색: 비교 횟수 2

[경우 2] 16, 13, 9, 7, 5, 4, 1 (내림차순)
- 순차 탐색: 비교 횟수 6
- 이진 탐색: 비교 횟수 2

순차 탐색일 경우 경우 1처럼 데이터가 오름차순으로 정렬되어 있다면 2회 만에 찾지만, 경우 2처럼 내림차순으로 정렬되어 있다면 6회 만에 찾는다. 반면, 이진 탐색은 정렬 기준에 따른 탐색 횟수의 편차가 적은 편이다. 이는 탐색 범위를 절반으로 줄이면서 진행하는 이진 탐색의 특성 때문이다.

데이터 개수가 적을 경우에는 탐색 횟수가 크게 차이 나지 않을 수 있지만, 데이터 개수가 많으면 선택하는 탐색 알고리즘에 따라 효율성에 큰 차이를 보일 수 있다.

[표 III-4 탐색 유형별 최대 교환 횟수] (단, n은 데이터 개수이다.)
- n = 10: 순차 탐색 10회 / 이진 탐색 4회
- n = 100: 순차 탐색 100회 / 이진 탐색 7회
- n = 1,000: 순차 탐색 1,000회 / 이진 탐색 10회
- n = 10,000: 순차 탐색 10,000회 / 이진 탐색 14회
- n = 1,000,000: 순차 탐색 1,000,000회 / 이진 탐색 20회

일반적으로 데이터가 정렬되어 있는 경우, 이진 탐색은 순차 탐색보다 훨씬 효율성이 좋다.
(그림 III-4 탐색 알고리즘의 효율성 그래프: 데이터 개수가 늘어날수록 순차 탐색의 비교 횟수는 급격히 증가하지만, 이진 탐색은 매우 완만하게 증가함)

[점검하기]
1. 탐색이란 나열된 데이터 중에서 원하는 데이터를 찾는 과정이다.
2. 순차 탐색은 데이터를 앞에서부터 차례대로 비교해서 원하는 데이터를 찾는 탐색 방법이다.
3. 이진 탐색은 탐색 범위를 반으로 줄여가며 원하는 데이터를 찾는 방법이다.
4. 데이터 개수가 많으면 탐색 알고리즘에 따라 효율성에서 큰 차이가 날 수 있다.
