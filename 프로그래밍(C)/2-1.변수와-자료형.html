Think & Objective

"컴퓨터는 숫자와 문자를 내부적으로 어떻게 구분하여 저장할까요?"

Section 1

C 프로그램의 첫걸음: 기본 뼈대 이해하기

처음 C언어 코드를 보면 마치 암호문 같아 보일 수 있습니다. 하지만 겁먹을 필요 없습니다! C 프로그램은 항상 정해진 기본 뼈대(구조) 안에서 작성됩니다. 아래의 가장 기초적인 "Hello, World" 출력 코드를 보며, 각 줄이 어떤 역할을 하는지 해부해 볼까요?

예제 기본 뼈대 코드
hello.c 
#include <stdio.h>

int main()
{
    printf("Hello, World\n");
    return 0;
}

#include <stdio.h>

"도구 상자 가져오기!" 화면에 글자를 보여주거나 키보드 입력을 받기 위해 필요한 기본 도구들을 프로그램에 포함시키는 주문입니다.

int main() { ... }

"프로그램의 현관문!" 컴퓨터가 프로그램을 실행할 때 가장 먼저 찾아서 들어가는 시작점입니다. 모든 코드는 이 { } 중괄호 안에 들어갑니다.

printf("Hello, World\n");

"화면에 출력해 줘!" 따옴표 안의 글자를 화면에 보여주는 명령어입니다. 끝에 있는 \n은 '엔터(줄바꿈)'를 의미합니다.

return 0;

"무사히 잘 끝났습니다!" 프로그램이 아무런 에러 없이 성공적으로 작업을 마쳤다는 것을 컴퓨터에게 알려주며 종료하는 의미입니다.

Section 3

컴퓨터 메모리의 구조와 자료형의 탄생

프로그램이 어떤 계산이나 작업을 하려면, 반드시 그 대상이 되는 데이터(숫자, 문자 등)를 메모리(작업 책상)에 올려두어야 합니다. 컴퓨터의 메모리는 1바이트(Byte) 크기의 수많은 작은 방들이 일렬로 늘어선 거대한 아파트와 같습니다. 컴퓨터는 데이터가 어느 방에 있는지 헷갈리지 않기 위해 각 방마다 0x1000과 같은 고유한 주소(Address)를 부여합니다.

0x1000이 뭐예요? — 메모리 주소 읽는 법

① 0x는 "16진수"라는 표시입니다.

우리는 평소에 10진수(0~9)를 씁니다. 컴퓨터는 주소를 표현할 때 16진수(0~9, A~F)를 씁니다. 지금은 "컴퓨터가 방 번호를 부르는 방식"이라고만 이해해도 충분합니다.

② 주소가 4씩 뛰는 이유는?

int는 4바이트(방 4칸)를 차지합니다. 따라서 다음 변수는 4칸 건너뛴 위치에서 시작합니다. 0x1000 → 0x1004 → 0x1008로 이동하는 것도 int 크기(4칸) 때문입니다.

0x1000 = 방 번호 4096 0x1004 = 방 번호 4100 (+4칸) 0x1008 = 방 번호 4104 (+4칸)

Memory View (1칸 = 1 Byte)

※ char(1칸)은 0x1000 → int(4칸)는 0x1004 → double(8칸)는 0x1008에서 시작합니다.

그래서 '자료형'과 '변수'가 필요합니다!

1. 자료형의 필요성: 위 도식을 보면 데이터의 종류와 크기에 따라 필요한 방의 개수가 다릅니다. 컴퓨터에게 "이 데이터는 방을 몇 칸이나 묶어서 쓸 것인지" 미리 알려주는 규칙이 바로 int, double 같은 자료형(Data Type)입니다.

2. 변수의 필요성: 만약 내 게임 캐릭터의 체력(100)을 가져오기 위해 매번 0x1004라는 기계적인 주소를 외워야 한다면 코딩은 악몽이 될 것입니다. 그래서 복잡한 주소 대신 hp, score처럼 사람이 읽기 쉬운 이름표를 붙여서 관리하는데, 이것이 바로 변수(Variable)입니다.

Section 4

자료형(Data Type) 완벽 가이드: 어떤 상자를 고를까?

"가장 큰 상자 하나만 만들어서 모든 데이터를 다 넣으면 안 될까요?"

만약 작은 지우개 하나를 보관하기 위해 거대한 이삿짐 박스를 쓴다면 공간 낭비가 심하겠죠? 컴퓨터의 메모리(RAM)는 한정되어 있습니다. 따라서 데이터의 종류와 크기에 맞게 가장 효율적인 크기의 방을 준비해야 합니다. C언어에서 가장 많이 쓰이는 3대장 자료형을 알아봅시다.

문자형 (char)

크기: 1 Byte

알파벳, 숫자, 기호 단 1개만 저장할 수 있는 가장 작은 상자입니다.

⚠️ 주의 규칙
반드시 ' ' (작은따옴표)로 감싸야 합니다. 쌍따옴표("")는 절대 안 됩니다! 
char grade = 'A';
char symbol = '#';

비밀: 컴퓨터는 문자도 숫자로 저장합니다!

컴퓨터는 'A'를 그 자체로 저장하는 게 아니라 숫자 65로 저장합니다. 각 문자에 고유 번호를 부여한 약속을 ASCII 코드라고 합니다. 섹션 2.5의 비트 시각화에서 65(= 01000001)를 본 것도 바로 이것이에요!

'A'=65
'B'=66
'a'=97
'0'=48

→ 이 때문에 char 상자에 숫자(0~127)를 직접 넣을 수도 있습니다. 섹션 8의 오버플로우 예제가 이 원리를 활용합니다.

정수형 (int)

크기: 4 Bytes

소수점이 없는 숫자를 저장합니다. 약 -21억부터 +21억까지의 수를 담을 수 있습니다.

💡 기억하세요
숫자는 따옴표 없이 그냥 적습니다. (따옴표를 붙이면 문자로 인식됨) 
int age = 20;
int money = -5000;

왜 ±21억일까요? (계산 원리)

4 Bytes → 32 Bits
2³² = 약 42억 가지
절반은 음수 → ±약 21억

방이 4개(4Byte)면 32개의 전구가 있고, 32개 전구가 만들 수 있는 조합의 절반이 양수, 절반이 음수입니다.

실수형 (double)

크기: 8 Bytes

소수점이 있는 정밀한 숫자를 저장합니다. (float형도 있지만, 현대에는 더 정밀한 double을 기본으로 씁니다.)

🎯 소수점 필수
반드시 소수점을 포함해서 적어주는 것이 좋습니다. 
double pi = 3.141592;
double height = 175.5;

Section 3.5

자료형의 메모리 크기, 눈으로 비교하기

자료형마다 메모리에서 차지하는 방의 개수가 다릅니다. 아래 다이어그램은 같은 메모리 공간을 기준으로 각 자료형이 얼마나 많은 방을 점유하는지 직접 눈으로 확인하는 시각 자료입니다. 큰 박스 안에 작은 칸 하나가 바로 메모리 1바이트(1칸)입니다.

미사용 공간

내 컴퓨터에서 직접 크기를 확인하는 방법: sizeof()

sizeof(자료형)을 사용하면 해당 자료형이 내 시스템에서 몇 Byte를 차지하는지 직접 출력할 수 있습니다.

#include <stdio.h>

int main() {
    printf("char  크기: %d Bytes\n", sizeof(char));   // 출력: 1
    printf("int   크기: %d Bytes\n", sizeof(int));    // 출력: 4
    printf("double크기: %d Bytes\n", sizeof(double)); // 출력: 8
    return 0;
}
char 크기: 1 Bytes
int 크기: 4 Bytes
double크기: 8 Bytes

참고: int와 같은 일부 자료형의 크기는 운영체제나 컴파일러 환경에 따라 다를 수 있습니다. 그래서 실제 크기가 궁금할 때는 sizeof()를 직접 사용하는 습관을 들이는 것이 좋습니다.

Section 5

변수 선언: 메모리 방에 이름표 붙이기

사람이 0x1000 같은 메모리 주소를 일일이 외울 수는 없습니다. 그래서 방을 확보할 때 사람이 알아보기 쉬운 이름표(변수명)를 붙입니다. 변할 수 있는 값이 들어가는 상자라는 뜻에서 변수(Variable)라고 부릅니다. 

int age; // "컴퓨터야, 정수(int)를 담을 4칸짜리 방을 확보하고 'age'라는 이름표를 붙여줘!"

⚠️ 선언 직후 (쓰레기값)

공간만 확보했을 뿐, 그 안에는 예전에 다른 프로그램이 쓰다 버린 알 수 없는 데이터가 들어있습니다.

age
?#!@ 쓰레기값
0x1000 ~ 0x1003

변수 이름 짓기 절대 규칙!

  • 영문자, 숫자, 밑줄(_)만 사용 가능 (특수문자 ❌)
  • 숫자로 시작할 수 없음 (예: 1st_score 금지)
  • 띄어쓰기 금지! (단어 연결은 my_score처럼)

Section 6

변수 할당: 준비된 상자에 값 넣기

준비된 변수 상자에 실제 데이터를 넣는 것을 할당(대입)이라고 합니다.
🚨 주의: C언어에서 = 기호는 '수학의 같다'가 아닙니다! "오른쪽의 값을 왼쪽 상자에 집어 넣어라"라는 뜻입니다. 선언과 할당을 동시에 하는 것을 초기화라고 부릅니다.

할당 예시
int age;
age = 20; // 할당

int score = 100; // 선언과 동시에 초기화!

✨ 할당 후의 메모리 상태

기존에 있던 쓰레기값은 덮어씌워져 사라지고, 우리가 넣은 20이 정확히 저장됩니다.

20
0x1000 ~ 0x1003

선언 → 할당 → 재할당의 전체 흐름을 단계별로 시뮬레이션합니다

앞에서 배운 변수 선언과 변수 할당을 합쳐서, 아래 코드가 순서대로 실행될 때 컴퓨터 메모리(RAM)의 상태가 어떻게 바뀌는지 살펴봅시다.

Section 6.5

모양이 다른 상자에 데이터를 넣는다면? (자료형 주의사항)

앞서 변수는 데이터를 담는 '맞춤형 상자'라고 배웠습니다. 그렇다면 만약 정수를 담기로 한 int 상자에 소수점이 있는 실수 데이터를 억지로 넣으면 어떻게 될까요? 컴퓨터는 상자의 모양에 맞추기 위해 데이터의 일부를 잘라내 버립니다.

잘못된 할당 예시
// 정수를 담는 int 방에 3.14라는 실수를 넣으려고 시도!
int myScore = 95.5;

// 컴퓨터: "이 상자는 소수점을 담을 공간이 없어! 잘라내자!"
// 실제 myScore 상자에는 '95'만 저장됩니다.

데이터 손실 (Truncation)

실수형 데이터(95.5)를 정수형 변수(int)에 넣으면, 소수점 이하의 값은 반올림되지 않고 무조건 버려집니다. 프로그래밍에서는 이를 데이터가 잘려 나갔다고 표현하며, 원치 않는 결과가 나올 수 있으므로 항상 자료형의 짝을 맞춰주는 것이 중요합니다.

Section 6.7

변하지 않는 값 — 상수(Constant)

한 번 정하면 절대 바꿀 수 없는 '자물쇠 달린 상자'

변수(Variable)가 '언제든지 값을 바꿀 수 있는 상자'라면, 상수(Constant)는 '한 번 값을 넣으면 절대로 바꿀 수 없는, 자물쇠가 채워진 상자'입니다. 수학의 원주율(π), 빛의 속도처럼 프로그램이 실행되는 동안 절대 변해서는 안 되는 값에 사용합니다.

절대 바뀌지 않습니다.

변수 vs 상수: 무엇이 다른가요?

변수 (Variable)

선언 키워드: int, double, char ...

score
100

✅ 언제든지 값을 바꿀 수 있습니다

int score = 100;
score = 150; // ✅ 가능!

상수 (Constant)

선언 키워드: const + 자료형

PI
3.14159

🔒 한 번 정하면 절대 바꿀 수 없습니다

const double PI = 3.14159;
PI = 3.0; // ❌ 컴파일 오류!

상수 선언 문법

const 자료형 이름 = 값 ;

상수 이름은 관례적으로 대문자와 밑줄(_)로 씁니다. (예: MAX_LEVEL, PI)

constants_example.c 
#include <stdio.h>

int main() {
    // 상수: 선언 이후 값을 바꿀 수 없습니다.
    const double PI = 3.14159;     // 원주율 (절대 불변)
    const int MAX_LEVEL = 99;      // 게임 최대 레벨 (고정)

    // 변수: 언제든지 값을 바꿀 수 있습니다.
    int myLevel = 1;               // 내 현재 레벨 (변할 수 있음)
    double radius = 5.0;           // 반지름 (변할 수 있음)

    printf("원주율: %.5lf\n", PI);
    printf("게임 최대 레벨: %d\n", MAX_LEVEL);
    printf("원의 넓이: %.2lf\n", PI * radius * radius);

    myLevel = 50;  // ✅ 변수는 값 변경 가능
    // MAX_LEVEL = 100; // ❌ 오류! 상수는 값 변경 불가

    printf("현재 레벨: %d / 최대 레벨: %d\n", myLevel, MAX_LEVEL);

    return 0;
}
원주율: 3.14159 게임 최대 레벨: 99 원의 넓이: 78.54 현재 레벨: 50 / 최대 레벨: 99

상수를 사용하면 좋은 이유

①
실수로 값을 바꾸는 버그를 방지합니다.
코드가 길어지면 실수로 PI를 덮어쓸 수 있습니다. const가 이를 차단합니다.
②
코드의 의미가 명확해집니다.
3.14159보다 PI가 훨씬 읽기 쉽습니다. 한 곳만 수정해도 전체 적용됩니다.

Section 7

화면에 출력하기: printf의 마법과 서식 지정자

변수에 담긴 값을 화면에 예쁘게 보여주려면 printf 함수를 사용합니다. C언어는 문자열 안에 변수의 값이 들어갈 '빈 의자(예약석)'를 미리 만들어 두어야 하는데, 이를 서식 지정자(Format Specifier)라고 부릅니다.

printf가 메모리에서 값을 가져오는 과정

printf( "나이: %d세\n" ,  age );
① 서식 지정자 "정수(int) 형식으로 읽어줘"
② 변수 이름 "이 상자의 값을 써줘"
age 상자를 찾아 값을 꺼냅니다
age
20
0x1000 ~ 0x1003 (int, 4Bytes)
꺼낸 값 20을 %d 자리에 채워 화면에 출력
나이: 20세

왜 자료형마다 다른 서식 지정자를 써야 할까요?

자료형마다 메모리에 값을 저장하는 방식(비트 패턴)이 다릅니다. int는 4바이트를 정수 형식으로 저장하고, double은 8바이트를 소수점 형식으로 저장합니다. printf는 메모리에서 값을 꺼낼 때 "몇 바이트를, 어떤 형식으로 해석할지"를 알아야 합니다. 그 역할을 하는 것이 바로 서식 지정자입니다.

%d → int
4바이트를
정수로 해석
%lf → double
8바이트를
실수로 해석
%c → char
1바이트를
문자로 해석

⚠️ int 변수에 %lf를 쓰거나, double 변수에 %d를 쓰면 엉뚱한 값이 출력됩니다! 반드시 짝을 맞추세요.

%d 정수 (int)
Decimal
%f / %lf 실수 (float / double)
꿀팁: %.2lf 라고 쓰면
소수점 둘째 자리까지만 출력!
%c 문자 (char)
Character

💡 핵심 문법: 여러 개의 변수 동시에 출력하기

서식 지정자의 개수와 뒤에 적어주는 변수의 개수는 반드시 일치해야 하며, 순서대로 1:1 매칭됩니다. 

int age = 20;
double height = 175.5;
char bloodType = 'A';

// %d 자리에는 age, %.1lf 자리에는 height, %c 자리에는 bloodType이 들어갑니다.
printf("나이: %d세\n키: %.1lfcm\n혈액형: %c형\n", age, height, bloodType);

키: 175.5cm
혈액형: A형

Section 7.5

변수의 진짜 매력: 저장된 값 변경하기 (재할당)

변수(Variable)라는 이름 자체가 '변할 수 있는 수'를 의미합니다. 한 번 상자에 값을 넣었더라도, 언제든지 새로운 값을 덮어씌울 수 있습니다. 초보자가 가장 헷갈려 하는 구문인 age = age + 1;을 통해 프로그래밍만의 독특한 규칙을 알아봅시다.

핵심 규칙: "항상 = (대입 연산자)의 오른쪽을 먼저 계산하라!"

1단계: 기존 상자의 값 읽기

age = 20 + 1;

2단계: 우측 수식 계산하기

age = 21;

3단계: 왼쪽 상자에 최종 저장

age 상자에 21 덮어씌우기! 
int coin = 100;    // 처음에 코인 상자에 100을 넣습니다.
coin = 50;         // 기존 100은 버려지고, 50으로 덮어씌워집니다. (재할당)

// 수학에서는 말이 안 되지만, 프로그래밍에서는 완벽한 문장입니다!
// 1. 오른쪽의 coin(현재 50) + 10을 먼저 계산하여 60을 만듭니다.
// 2. 그 60을 다시 왼쪽의 coin 상자에 집어넣습니다.
coin = coin + 10;

printf("현재 코인: %d\n", coin); // 결과: 60 출력

수학에서의 = 기호는 '양쪽이 같다'는 뜻이지만, 프로그래밍에서는 '오른쪽 결과를 왼쪽 상자에 집어넣어라'라는 명령입니다. 이 원리를 이해하면 앞으로 나올 모든 코드를 쉽게 읽어낼 수 있습니다.

Section 8

심화: 상자가 넘치면 어떻게 될까? (오버플로우)

이 내용은 심화 학습입니다. 처음 학습할 때는 건너뛰어도 괜찮습니다.

char형은 1바이트(방 1칸) 크기이므로 숫자를 넣을 경우 최대 127(또는 255)까지만 담을 수 있습니다. 자동차의 주행거리 계기판이 99,999km에서 1km를 더 달리면 00,000km로 돌아가듯, 변수도 최댓값을 넘어가면 가장 작은 값(최솟값)으로 빙글 돌아가버립니다. 이를 오버플로우(Overflow)라고 부릅니다.

섹션 4에서 char가 'A'(= 숫자 65)를 저장할 수 있다고 배웠죠? 같은 원리로 char smallBox = 127;처럼 직접 숫자를 넣을 수도 있습니다. 이때 최댓값을 초과하면 오버플로우가 발생합니다. 

char smallBox = 127; // char가 가질 수 있는 최댓값
smallBox = smallBox + 1; // 1을 더하면 128이 될까?

printf("%d\n", smallBox); // 결과는 -128 (최솟값으로 순환)
넘침!

오버플로우 현상

학습 포인트: 프로그래밍을 할 때는 "내가 다루려는 데이터의 크기가 얼마나 될까?"를 예상하고, 그에 맞는 넉넉한 크기의 자료형(예: int) 상자를 준비하는 것이 매우 중요합니다.

총정리: 나만의 RPG 캐릭터 스테이터스 만들기

지금까지 배운 기본 뼈대, 자료형, 상수, 변수 선언, 할당, 재할당, printf를 모두 합쳐서 하나의 완벽한 C 프로그램을 만들어 보겠습니다.

character_status.c 
#include <stdio.h>  // 1. 도구 상자 포함

int main() {        // 2. 프로그램 시작점

    // 3. 상수 선언 (절대 변하지 않는 게임 규칙!)
    const int MAX_LEVEL = 99;      // 최대 레벨 (고정값)
    const double POWER_BONUS = 1.5; // 레벨업 공격력 증가 배수 (고정값)

    // 4. 변수 선언 및 초기화 (알맞은 상자 고르기)
    char rank = 'S';             // 문자형 (1Byte, 홑따옴표 사용)
    int level = 10;              // 정수형 (4Bytes)
    double attackPower = 100.0;  // 실수형 (8Bytes)

    // 5. 화면에 첫 상태 출력하기
    printf("=== 캐릭터 초기 상태 ===\n");
    printf("등급: %c\n", rank);
    printf("레벨: %d / 최대: %d\n", level, MAX_LEVEL);
    printf("공격력: %.1lf\n\n", attackPower);

    // 6. 변수 값 변경 (재할당: 사냥을 해서 레벨업을 했다고 가정해봅시다!)
    printf("... 몬스터를 사냥했습니다! 레벨업! ...\n\n");
    level = level + 1;                          // 기존 10 + 1 = 11
    attackPower = attackPower * POWER_BONUS;    // 상수를 활용한 계산!

    // 7. 변경된 상태 다시 출력하기
    printf("=== 캐릭터 성장 후 상태 ===\n");
    printf("등급: %c\n", rank);
    printf("레벨: %d (Level Up!)\n", level);
    printf("공격력: %.1lf (Power Up!)\n", attackPower);

    return 0; // 8. 프로그램 정상 종료
}
=== 캐릭터 초기 상태 === 등급: S 레벨: 10 / 최대: 99 공격력: 100.0
... 몬스터를 사냥했습니다! 레벨업! ...
=== 캐릭터 성장 후 상태 === 등급: S 레벨: 11 (Level Up!) 공격력: 150.0 (Power Up!)
