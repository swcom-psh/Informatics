Chapter 2. C 프로그래밍의 기초

4. 제어문 (Control Statement), 반복 구조

프로그램의 흐름을 내 마음대로! 조건에 따라 다른 길을 가거나, 똑같은 작업을 반복하는 제어 구조를 단계적으로 마스터합니다.

학습 목표 (Objective)

  • for, while, do-while의 차이를 이해하고 상황에 맞게 선택할 수 있다.
  • break와 continue로 반복 흐름을 제어할 수 있다.
  • 중첩 반복문(반복문 안의 반복문)을 작성하고 이해할 수 있다.

순차 구조

위에서 아래로 정해진 순서에 따라 차례대로 코드가 실행되는 가장 기본적인 흐름 구조입니다.

선택 구조

주어진 조건의 참·거짓에 따라 서로 다른 코드를 실행하는 분기 구조입니다. (if, switch)

반복 구조

특정 조건이 만족될 때까지 동일한 코드를 계속 반복 실행하여 작업을 자동화합니다. (for, while)

반복문(Iteration)의 마법

동일한 작업을 수천, 수만 번 직접 타이핑할 필요가 없습니다. 반복문은 정해진 횟수나 특정 조건이 만족될 때까지 컴퓨터가 대신 코드를 반복해서 실행해 주는 프로그래머의 가장 강력한 무기입니다.

반복문이 없다면? (코드 100번 쓰기)

1printf("Hello\n");
2printf("Hello\n");
3printf("Hello\n");
97번 더 직접 작성해야 함... 😭

반복문을 쓴다면? (단 3줄!)

1for (int i = 0; i < 100; i++) {
2printf("Hello\n");
3}
생산성 10000% 상승

Section 1

for 문

반복 횟수가 명확할 때 사용 (예: 10번 반복)

횟수 중심

for문은 어떤 순서로 움직일까?

for ( ① 초기식 ; ② 조건식 ; ④ 증감식 ) {
    ③ 실행문 ;
}
  • 1

    초기식: 반복 시작 시 딱 한 번만 실행 (i = 1)

  • 2

    조건식: 참이면 진입, 거짓이면 즉시 종료 (i <= 10)

  • 3

    실행문: 조건이 참일 때 실행할 메인 코드

  • 4

    증감식: 실행문이 끝나면 변수 업데이트 후 다시 ②번으로 (i++)

기초 예제 for_basic.c 
#include <stdio.h>

int main() {
    int sum = 0;
    // for(초기식; 조건식; 증감식)
    for (int i = 1; i <= 10; i++) {
        sum = sum + i; // 1+2+3+...+10
    }
    printf("1~10 합: %d\n", sum);
    return 0;
}
1~10 합: 55
응용 예제 103.c (구구단 2단 출력) 
#include <stdio.h>

int main() {
    int i;
    // i가 1부터 9까지 1씩 증가하며 반복
    for (i = 1; i < 10; i++) {
        printf("2 x %d = %d\n", i, 2 * i);
    }
    return 0;
}
2 x 1 = 2
2 x 2 = 4
2 x 3 = 6  ... (중략) ...
2 x 9 = 18
👀 눈으로 보는 반복문: 103.c 구구단 변수 추적표 (클릭)

초보자라면 코드를 눈으로만 보지 말고, 아래처럼 표를 그려서 변수가 어떻게 변하는지 추적(Trace)해 보는 연습이 꼭 필요합니다.

반복 회차 i의 값 조건식 (i < 10) 출력 결과 증감식
1회차 1 참 (1 < 10) 2 x 1 = 2 i++ → i=2
2회차 2 참 (2 < 10) 2 x 2 = 4 i++ → i=3
. . . (3~8회차 생략) . . .
9회차 9 참 (9 < 10) 2 x 9 = 18 i++ → i=10
10회차 10 거짓 (10 < 10) 실행 안 됨 (종료) -

학생 단골 실수: for문 끝에 세미콜론(;)을 붙이지 마세요!

조건식 끝에 무심코 ;을 붙이면, 컴퓨터는 '아무것도 안 하고' 반복이 끝난 것으로 인식합니다. 결과적으로 아래 중괄호 안의 코드는 단 1번만 실행되는 치명적인 버그가 발생합니다.

for (int i = 1; i <= 5; i++); // ← 치명적 실수! 이 줄에서 반복이 끝남
{
    printf("안녕하세요\n"); // 이 줄은 딱 1번만 실행됨!
}

Section 2

while 문

특정 조건이 유지되는 동안 계속 반복 (예: 게임이 끝날 때까지)

조건 중심

while문은 어떤 순서로 움직일까?

while ( ① 조건식 ) {
    ② 실행문 ;
    ③ 증감식 ; ← 직접 써야 함!
}
  • 1

    조건식: 매번 먼저 확인합니다. 참이면 진입, 거짓이면 즉시 종료.

  • 2

    실행문: 조건이 참일 때 실행됩니다. 끝나면 다시 ①번으로 돌아가 확인합니다.

  • 3

    증감식: for문과 달리 반복문 본문 안에 직접 작성해야 합니다. (빠뜨리면 무한루프!)

기초 예제 while_basic.c 
#include <stdio.h>

int main() {
    int i = 1, sum = 0;
    // 조건식이 참인 동안 반복
    while (i <= 10) {
        sum += i;
        i++;  // ← 이 줄이 없으면 무한루프!
    }
    printf("1~10 합: %d\n", sum);
    return 0;
}
1~10 합: 55
응용 예제 104-1.c (0 입력 시 종료) 
#include <stdio.h>

int main() {
    int answer = 1;
    // 0이 입력될 때까지 계속 반복
    while (answer != 0) {
        printf("숫자를 입력하세요 (0을 누르면 종료): ");
        scanf("%d", &answer);
        printf("%d 입력\n", answer);
    }
    return 0;
}
실행 예시 (3, 7, 0 순서로 입력):
숫자를 입력하세요 (0을 누르면 종료): 3↵
3 입력
숫자를 입력하세요 (0을 누르면 종료): 7↵
7 입력
숫자를 입력하세요 (0을 누르면 종료): 0↵
0 입력 (← 조건 거짓, 종료)

💡 사용자가 3, 7, 0을 차례대로 입력했다고 가정합니다.

반복 회차 조건식 (answer != 0) 사용자 입력 출력 결과 answer 값
시작 전 - - - 1 (초기값)
1회차 참 (1 != 0) 3 입력 3 입력 3
2회차 참 (3 != 0) 7 입력 7 입력 7
3회차 참 (7 != 0) 0 입력 0 입력 0
4회차 거짓 (0 != 0) 실행 안 됨 → 반복문 종료

무한 루프(Infinite Loop) 주의!

while문 블록 안에 조건을 변화시키는 코드(i++)를 빼먹으면, 조건이 영원히 참이 되어 프로그램이 멈추지 않고 영원히 돌게 됩니다.

int i = 0;
while (i < 5) {
    printf("안녕하세요\n");
    // i++; ← 이 코드가 빠지면 i는 계속 0 → 무한 반복!
}

Section 2.5

어떤 반복문을 선택해야 할까?

for문과 while문은 모두 같은 작업을 할 수 있습니다. 하지만 상황에 따라 더 적합한 도구가 있습니다. 아래 기준을 참고하세요.

for

반복 횟수가 정해진 경우

몇 번 반복할지 미리 알 때 사용합니다. 초기식·조건식·증감식이 한 줄에 모여 있어 가독성이 좋습니다.

// 1부터 10까지 10번 반복
for (i = 1; i <= 10; i++) { ... }
while

조건이 만족될 때까지

몇 번 반복할지 모르고, 조건이 거짓이 될 때까지 반복할 때 사용합니다. 사용자 입력, 게임 루프 등에 적합합니다.

// 0 입력 전까지 반복
while (answer != 0) { ... }
do-while

무조건 1번은 실행

조건 확인 전에 반드시 1번은 실행해야 할 때 사용합니다. "먼저 입력받고 나서 검사"하는 패턴에 적합합니다.

// 일단 실행 후 조건 검사
do { scanf(...); } while (...);
반복문 조건 검사 시점 최소 실행 횟수 적합한 상황
for 실행 전 (선검사) 0번 (조건이 처음부터 거짓이면) 횟수가 정해진 반복
while 실행 전 (선검사) 0번 (조건이 처음부터 거짓이면) 조건 기반 반복
do-while 실행 후 (후검사) 최소 1번 (무조건) 먼저 실행 후 검사

Section 3

do-while 문

무조건 1번 실행 후 조건 검사

do-while 문은 while문과 비슷하지만, 조건의 참·거짓과 상관없이 먼저 한 번은 실행한 뒤에 조건을 검사한다는 점이 다릅니다.

do-while문은 어떤 순서로 움직일까?

do {
    ① 실행문 ;
} while ( ② 조건식 ); ← 세미콜론(;) 필수!
  • 1

    실행문: 반복문 안의 코드를 무조건 한 번 실행합니다. 조건 검사 없이 진행됩니다.

  • 2

    조건식: 실행문이 끝난 후 확인합니다. 참이면 다시 실행문으로 돌아가 반복하고, 거짓이면 종료합니다.

핵심 통찰: while과 do-while은 언제 다를까?

앞서 배운 104-1.c (while문) 코드를 보면 int answer = 1;처럼 시작부터 억지로 1을 넣어주었습니다. 만약 0을 넣으면 시작하자마자 조건이 거짓이 되어 입력을 단 한 번도 받지 못하기 때문입니다.

하지만 do-while문은 무조건 1번은 실행하므로 초기값을 억지로 세팅할 필요 없이, "일단 입력받고 나서 검사해야 하는 상황"에서 훨씬 자연스럽습니다.

예제 do~while문 (105-1.c)
105-1.c 
#include <stdio.h>
int main()
{
    int answer;
    do {
        scanf("%d", &answer);
        printf("%d 입력\n", answer);
    } while (answer != 0);  // ← 세미콜론 필수!
    return 0;
}
실행 예시 (5, 0 순서로 입력):
5↵
5 입력
0↵
0 입력 (← 조건 거짓, 종료)

💡 사용자가 5, 0을 순서대로 입력했다고 가정합니다. 조건 검사가 실행 이후에 일어난다는 점을 주목하세요.

반복 회차 사용자 입력 출력 결과 조건식 (answer != 0)
1회차 (무조건 실행) 5 5 입력 참 → 반복
2회차 0 0 입력 거짓 → 종료
심화 예제 소문자 → 대문자 변환 (105-2.c)
105-2.c 
#include <stdio.h>
int main()
{
    char ch;
    do {
        scanf(" %c", &ch);
        printf("소문자 %c는 대문자 %c입니다.\n", ch, ch - 32);
    } while (ch != 'q');  // q를 입력하면 종료
    return 0;
}
실행 예시:
a↵
소문자 a는 대문자 A입니다.
q↵
소문자 q는 대문자 Q입니다. (← ch=='q', 조건 거짓, 종료)

잠깐! 문자에 숫자를 빼나요? (ch - 32)

컴퓨터는 문자를 고유 숫자(아스키코드)로 저장합니다. 소문자 'a'는 97, 대문자 'A'는 65입니다. 소문자와 대문자는 항상 32만큼 차이가 나기 때문에, 소문자에서 32를 빼면 대문자가 됩니다!

Section 4

반복 흐름 제어: break & continue

반복문을 실행하다가 강제로 빠져나오거나, 특정 부분만 건너뛰고 싶을 때 break와 continue를 사용합니다.

break — "즉시 탈출"

조건을 만족하면 반복문을 그 즉시 완전히 종료하고 반복문 바깥으로 나갑니다.

for (int i = 1; i <= 10; i++) {
    if (i == 5) break;  // 5가 되면 탈출
    printf("%d ", i);
}
// 결과: 1 2 3 4
1 2 3 4

continue — "현재 회차 건너뛰기"

이어지는 명령문을 무시하고 다음 반복 회차로 넘어갑니다. 반복 자체는 계속됩니다.

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue; // 3은 건너뜀
    printf("%d ", i);
}
// 결과: 1 2 4 5
1 2 4 5

break vs continue 실행 흐름 비교

break 실행 흐름 (i==5에서 break)

i=1 → 출력 → i=2

i=2 → 출력 → i=3

i=3 → 출력 → i=4

i=4 → 출력 → i=5

i=5 → break → ★ 반복문 완전 종료

↓ 반복문 이후 코드 실행

continue 실행 흐름 (i==3에서 continue)

i=1 → 출력 → i=2

i=2 → 출력 → i=3

i=3 → continue → ★ 나머지 건너뜀 → i=4

i=4 → 출력 → i=5

i=5 → 출력 → 종료

앞선 선택 구조 단원에서 "C언어는 0이 아닌 모든 수를 참(True)으로 본다"고 배웠습니다. 따라서 while (1)이라고 적으면 조건이 "무조건 참"이 되어 의도적인 무한 루프가 만들어집니다.

무한 루프를 만들 때는 반드시 내부에 if 조건문과 break;를 짝꿍처럼 결합하여 "탈출구"를 만들어 주어야 합니다.

예제 break문: while(1)로 무한루프 + 탈출 (106.c)
106.c 
#include <stdio.h>
int main()
{
    int answer;
    while (1)  // while(1) = 무한루프
    {
        scanf("%d", &answer);
        if (answer == 0)
        {
            break;  // 0 입력 시 탈출!
        }
        else {
            printf("%d 입력\n", answer);
        }
    }
    return 0;
}
실행 예시 (10, 20, 0 순서로 입력):
10↵
10 입력
20↵
20 입력
0↵
(0 입력 → break → 프로그램 종료)
👀 눈으로 보는 반복문: 106.c 변수 추적표 (클릭)

💡 사용자가 10, 20, 0을 순서대로 입력했다고 가정합니다.

반복 회차 answer 입력값 if (answer == 0) 출력 결과 / 동작
1회차 10 거짓 → else 실행 10 입력
2회차 20 거짓 → else 실행 20 입력
3회차 0 참 → break! 즉시 반복문 탈출
예제 continue: for문과 배수 세기 (107-1.c)
107-1.c 
#include <stdio.h>
int main()
{
    int i, count = 0, answer;
    scanf("%d", &answer);
    for (i = 1; i < 101; i++)
    {
        if (i % answer != 0)
        {
            continue;  // 배수가 아니면 건너뜀
        }
        count++;  // 배수일 때만 실행
    }
    printf("1~100 중 %d의 배수는 %d개입니다.",
        answer, count);
    return 0;
}
실행 예시 (answer = 3 입력 시):
3↵
1~100 중 3의 배수는 33개입니다.
👀 눈으로 보는 반복문: 107-1.c 변수 추적표 (클릭)

💡 answer=3 입력, i=1~5까지만 추적합니다.

i 값 배수 확인 (i % 3 != 0) 실행 흐름 count 값
1 참 (1%3=1≠0) continue → count++ 건너뜀 0
2 참 (2%3=2≠0) continue → count++ 건너뜀 0
3 거짓 (3%3=0) count++ 실행! 1
4 참 (4%3=1≠0) continue → 건너뜀 1
5 참 (5%3=2≠0) continue → 건너뜀 1

⚠️ while문에서 continue 쓸 때 주의: 무한루프 함정!

for문과 달리 while문은 증감식(i++)을 보통 블록 맨 아래에 적습니다. 만약 증감식 위에 continue;가 있으면 어떻게 될까요? 증감식을 만나기도 전에 위로 튕겨 올라가 버리기 때문에, 변수 값이 영원히 변하지 않고 무한 루프에 빠지게 됩니다.
이를 방지하기 위해 107-2.c 예제에서는 아예 조건식 안에 while (i++ < 10)처럼 조건 검사와 증감을 하나로 합쳐서 작성했습니다. 아주 영리한 코드죠!

❌ 무한루프 (버그)

int i = 0;
while (i < 10) {
  if (i == 5) continue;
  printf("%d ", i);
  i++; // continue로 인해 실행 안 됨!
}

✅ 올바른 코드 (107-2.c 방식)

int i = 0;
while (i++ < 10) {
  if (i == 5) continue;
  printf("%d ", i);
}
// 조건식 안에서 i++를 처리!
예제 while + continue (107-2.c) — 조건식에서 증감 처리
107-2.c 
#include <stdio.h>
int main()
{
    int i = 0;
    // while 조건식 안에서 i++를 처리 → continue가 i++를 건너뛰지 못함!
    while (i++ < 10)
    {
        if (i % 2 == 0)
        {
            continue;  // 짝수는 건너뜀
        }
        printf("%d ", i);  // 홀수만 출력
    }
    return 0;
}

Section 5

중첩 반복문 (Nested Loop)

반복문 안에 또 다른 반복문

반복문 안에 또 다른 반복문을 넣는 중첩 제어 구조입니다. 논리적 사고가 필요하지만, '시계바늘'을 떠올리면 이해하기 쉽습니다.

시계의 분침과 초침의 원리

시계의 분침(바깥쪽 반복문)이 딱 1칸 움직이려면, 초침(안쪽 반복문)은 60칸을 팽그르르 돌아야 합니다. 바깥쪽 반복문이 1번 실행될 때마다, 안쪽 반복문은 자신의 처음부터 끝까지 완전히 다 돌아야만 바깥쪽이 다음 회차로 넘어갑니다.

중첩 반복문 실행 순서 (i=1~3, j=1~3 기준)

i=1→j=1→j=2→j=3→ 안쪽 루프 종료
i=2→j=1→j=2→j=3→ 안쪽 루프 종료
i=3→j=1→j=2→j=3→ 안쪽 루프 종료
→ 바깥 루프 종료 (총 3×3 = 9회 실행)
예제 중첩 for문: 직각 삼각형 별 찍기
nested_for.c 
#include <stdio.h>

int main() {
    // 바깥쪽 루프: 줄 번호 (분침)
    for (int i = 1; i <= 3; i++) {

        // 안쪽 루프: 각 줄에 별 찍기 (초침)
        for (int j = 1; j <= i; j++) {
            printf("*");
        }
        printf("\n"); // 안쪽 루프가 끝나면 줄바꿈
    }
    return 0;
}

바깥쪽 (i) 안쪽 (j) 범위 실행 흐름 출력 누적
i=1 j는 1~1 (1번) j=1: * 출력 *↵
안쪽 종료 → \n

j=2: * 출력
안쪽 종료 → \n
i=3 j는 1~3 (3번) j=1: * 출력 ***↵
j=2: * 출력
j=3: * 출력
안쪽 종료 → \n
응용 예제 중첩 for문: 구구단 전체 (2~9단)
gugu_all.c 
#include <stdio.h>

int main() {
    // 바깥 루프: 단 (2단 ~ 9단)
    for (int i = 2; i <= 9; i++) {
        printf("=== %d단 ===\n", i);
        // 안쪽 루프: 각 단의 1~9
        for (int j = 1; j <= 9; j++) {
            printf("%d x %d = %d\n", i, j, i * j);
        }
    }
    return 0;
}

Self-Check

자가점검 — 스스로 답할 수 있나요?

책이나 노트를 보지 않고 답할 수 있으면 이 장의 학습이 완료된 것입니다.

Q1

반복문은 왜 필요한가요? 반복문이 없다면 어떻게 해야 하나요?

힌트: 동일한 코드 반복 작성 → 반복문으로 자동화

Q2

for문과 while문은 어떻게 다른가요? 각각 어떤 상황에 사용하나요?

힌트: 횟수 vs 조건 중심, 초기/증감식 위치

Q3

do-while문이 while문과 다른 점은 무엇인가요? 언제 더 유용한가요?

힌트: 조건 검사 시점, 최소 실행 횟수

Q4

break와 continue의 차이는 무엇인가요? for(i=1; i<=5; i++) { if(i==3) break; printf("%d",i); }의 출력은?

힌트: 탈출 vs 건너뜀

Q5

중첩 반복문에서 바깥쪽 반복문이 1회 실행될 때, 안쪽 반복문은 몇 번 실행되나요?

힌트: 시계의 분침과 초침 비유

정답 확인 (클릭)

A1. 동일한 작업을 여러 번 자동으로 실행하기 위해 필요합니다. 반복문 없이는 같은 코드를 n번 직접 복사해야 합니다.

A2. for문은 반복 횟수가 정해졌을 때, while문은 조건이 만족될 때까지 반복할 때 사용합니다. for문은 초기식·증감식이 괄호 안에 있고, while문은 증감식을 본문 안에 직접 써야 합니다.

A3. do-while은 조건 검사 전에 반드시 1번은 실행합니다. "일단 입력받고 나서 검사"하는 패턴(사용자 입력 검증 등)에 더 자연스럽습니다.

A4. break는 반복문 자체를 완전히 종료하고, continue는 현재 회차만 건너뜁니다. 위 코드의 출력: 1 2 (i==3에서 탈출, 4·5는 실행 안 됨)

A5. 안쪽 반복문은 처음부터 끝까지 완전히 한 바퀴 돕니다. 바깥쪽 3번 × 안쪽 3번 = 총 9번 실행됩니다.

반복 구조 핵심 요약

  • 용도에 따른 반복문 선택횟수가 정해진 반복은 for문, 특정 조건이 유지되는 동안은 while문, 최소 1회 실행이 필요하면 do-while문을 사용합니다.
  • 흐름 제어 (break와 continue)break는 반복문을 즉시 탈출하고, continue는 현재 회차를 건너뛰고 다음 반복으로 이동합니다.
  • 중첩 반복문 (시계 원리)바깥쪽 루프가 1회 실행될 때마다 안쪽 루프는 처음부터 끝까지 완전히 한 바퀴 돕니다. (분침 1칸 = 초침 1바퀴)
