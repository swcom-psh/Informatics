Chapter 2. C 프로그래밍의 기초

3. 연산자 (Operator)

데이터를 가공하고 계산하기 위한 연산자의 종류를 배우고, 우선순위와 결합 법칙을 통해 복합적인 수식을 이해합니다.

Think & Objective

"컴퓨터는 연산의 순서를 어떻게 결정할까요?"

  • 산술, 비교, 논리, 비트 등 다양한 연산자의 의미를 알고 적절하게 사용할 수 있다.
  • 연산자 우선순위를 이해하여 복잡한 수식을 올바르게 해석할 수 있다.
  • 형변환(캐스팅)의 필요성을 이해하고 실수 나눗셈을 올바르게 처리할 수 있다.

Section 1

산술 연산자 (Arithmetic)

가장 기본이 되는 연산자로, 수학의 사칙연산과 유사합니다. C언어만의 특별한 점은 정수 나눗셈의 소수점 버림과 나머지 연산(%)입니다.

핵심 포인트: / 와 %

  • / (나누기): 정수끼리 나누면 결과도 정수. 소수점은 버림. (5 / 2 = 2)
  • % (나머지): 나눈 후 남은 값. (5 % 2 = 1) → 짝/홀수 판별에 핵심!
💡 나머지 활용: 숫자 % 2 == 0 이면 짝수, == 1이면 홀수

연산 우선순위 (산술)

수학과 동일합니다. *, /, % → +, - 순서로 계산합니다.

// 4 + 3 * 5 에서 3*5 먼저!

4 + 3 * 5 = 4 + 15 = 19

(4 + 3) * 5 = 7 * 5 = 35 ← 괄호로 순서 변경

Step 1
기초 사칙연산 + 나머지
arithmetic.c 
#include <stdio.h>

int main() {
    int a = 10, b = 3;

    printf("덧셈: %d + %d = %d\n", a, b, a + b);    // 13
    printf("뺄셈: %d - %d = %d\n", a, b, a - b);    // 7
    printf("곱셈: %d * %d = %d\n", a, b, a * b);    // 30
    printf("나눗셈: %d / %d = %d\n", a, b, a / b);  // 3  ← 소수점 버림!
    printf("나머지: %d %% %d = %d\n", a, b, a % b); // 1  ← 10 = 3*3 + 1

    return 0;
}
덧셈: 10 + 3 = 13
뺄셈: 10 - 3 = 7
곱셈: 10 * 3 = 30
나눗셈: 10 / 3 = 3
나머지: 10 % 3 = 1
예제
연산자 우선순위 (86.c)
86.c 
#include <stdio.h>
int main()
{
    int result = 4 + 3 * 5;
    // 3*5=15, 4+15=19
    printf("%d", result);
    return 0;
}
19
곱셈(*)이 덧셈(+)보다 먼저 계산 → 3×5=15 먼저, 그 후 4+15=19
예제
5칙 연산 (87-1.c)
87-1.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5;
    // i+j=18  i-j=8  i*j=65
    // i/j=2(몫)  i%j=3(나머지)
    printf("%d %d %d %d %d",
        i+j, i-j, i*j, i/j, i%j);
    return 0;
}

Section 2

대입 및 복합 대입 연산자

단순히 값을 저장하는 = 외에도, 기존 값에 연산을 동시에 수행하는 복합 대입 연산자가 있습니다. 코드를 훨씬 짧고 읽기 쉽게 만들어 줍니다.

복합 대입 연산자 전체 표

a += b=a = a + b
a -= b=a = a - b
a *= b=a = a * b
a /= b=a = a / b
a %= b=a = a % b
예제
대입 연산 (87-2.c)
87-2.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5;
    i = j;  // j의 값(5)을 i에 복사
    // i가 원래 가진 13은 지워짐!
    printf("%d\n", i);
    return 0;
}
5
= 는 "같다"가 아니라 "오른쪽을 왼쪽에 넣어라"입니다.
Step 2
복합 대입 연산 실습
compound_assign.c 
#include <stdio.h>

int main() {
    int score = 10;

    score += 5;  // score = 10 + 5 = 15
    printf("+=  결과: %d\n", score); // 15

    score *= 2;  // score = 15 * 2 = 30
    printf("*=  결과: %d\n", score); // 30

    score -= 10; // score = 30 - 10 = 20
    printf("-=  결과: %d\n", score); // 20

    score %= 6;  // score = 20 % 6 = 2
    printf("%%=  결과: %d\n", score); // 2

    return 0;
}
+=  결과: 15
*= 결과: 30
-= 결과: 20
%=  결과: 2

Section 3

증감 연산자 (++, --)

값을 1씩 늘리거나 줄이는 단축 연산자입니다. 단독으로 쓸 때는 앞뒤 위치가 결과에 영향이 없지만, 다른 연산과 섞이면 실행 순서가 달라집니다.

전위(++a) vs 후위(a++) — 핵심 차이

++a (전위) — "선불"

먼저 1을 더하고, 증가된 값을 사용합니다.

int a = 10;

int result = ++a;

// a = 11, result = 11

a++ (후위) — "후불"

현재 값을 먼저 사용하고, 나중에 1을 더합니다.

int a = 10;

int result = a++;

// result = 10, 그 후 a = 11

🚨 단독 사용 시: a++; 와 ++a;는 결과가 완전히 같습니다. 차이는 result = a++처럼 대입과 섞일 때만 발생합니다.

예제
전위 vs 후위 연산의 차이 (88.c)
88.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5, temp,
        a = 10, b = 20, c = 30, d = 40;
    temp = ++i; printf("%d %d\n", temp, i); // 전위: i먼저 14, temp=14
    temp = i++; printf("%d %d\n", temp, i); // 후위: temp=14먼저, i→15
    temp = --j; printf("%d %d\n", temp, j); // 전위: j먼저 4, temp=4
    temp = j--; printf("%d %d\n", temp, j); // 후위: temp=4먼저, j→3
    ++a; b++; --c; d--;
    printf("%d %d %d %d", a, b, c, d);     // 단독사용: 전후위 차이없음
    return 0;
}
14 14
14 15
4 4
4 3
11 21 29 39
한 줄씩 따라가는 풀이 (i=13, j=5)
  1. temp = ++i (전위): i를 먼저 14로 만든 후 temp에 14를 대입. → temp=14, i=14 → 출력: 14 14
  2. temp = i++ (후위): temp에 현재 i(14)를 먼저 대입한 후, i를 15로 증가. → temp=14, i=15 → 출력: 14 15
  3. temp = --j (전위): j를 먼저 4로 만든 후 temp에 4를 대입. → temp=4, j=4 → 출력: 4 4
  4. temp = j-- (후위): temp에 현재 j(4)를 먼저 대입한 후, j를 3으로 감소. → temp=4, j=3 → 출력: 4 3
  5. ++a; b++; --c; d--: 단독 사용 → 전후위 결과 동일. a=11, b=21, c=29, d=39 → 출력: 11 21 29 39

Section 4

관계(비교) 연산자

두 값을 비교하는 연산자입니다. C언어는 "True(참)"나 "False(거짓)" 대신 숫자로 대답합니다. 맞으면 1 (참), 틀리면 0 (거짓).

워밍업: 컴퓨터는 참/거짓을 숫자로 표현합니다

printf("%d", 10 > 5);
// 10이 5보다 크므로 → 1 출력 (참)
printf("%d", 10 == 5);
// 10과 5는 같지 않으므로 → 0 출력 (거짓)

관계 연산자 전체 표

==양쪽이 같으면 1
!=양쪽이 다르면 1
>왼쪽이 크면 1
<왼쪽이 작으면 1
>=크거나 같으면 1
<=작거나 같으면 1
예제
관계 연산자 (89-1.c)
89-1.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5;
    printf("%d %d %d %d %d %d",
        i<j, i>j, i<=j, i>=j, i!=j, i==j);
    return 0;
}
0 1 0 1 1 0
13<5=0, 13>5=1, 13≤5=0, 13≥5=1, 13≠5=1, 13=5=0

Section 5

논리 연산자 & 삼항 연산자

논리 연산자 (조건 결합)

&& (AND) — "그리고"

양쪽이 모두 참(1)일 때만 참. 하나라도 거짓이면 거짓.

1&&1=1, 1&&0=0, 0&&0=0

|| (OR) — "또는"

둘 중 하나라도 참이면 참. 둘 다 거짓이어야 거짓.

1||1=1, 1||0=1, 0||0=0

! (NOT) — "아니다"

참이면 거짓으로, 거짓이면 참으로 뒤집습니다.

!1=0, !0=1, !(0이 아닌 수)=0

예제
논리 연산자 (90-1.c)
90-1.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5, temp;
    // AND: 둘 다 참? / 하나라도 거짓?
    printf("%d %d\n", i==13 && j==5, i==13 && j==4);
    // OR: 하나라도 참? / 둘 다 거짓?
    printf("%d %d\n", i==13 || j==4, i==12 || j==4);
    // NOT: 뒤집기
    temp = i; printf("%d ", !temp); // !13=0
    temp = 0; printf("%d",  !temp); // !0=1
    return 0;
}
1 0
1 0
0 1
⚡ 주의
단락 회로 평가 (Short-Circuit) — 90-2.c
90-2.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5;
    // i < 10은 거짓(0) → &&이므로 ++j는 아예 실행되지 않음!
    if (i < 10 && ++j > 3) { }
    printf("%d %d\n", i, j); // j가 5 그대로!
    return 0;
}
13 5
A && B (AND):

A가 거짓(0)이면 B는 아예 평가하지 않습니다. 어차피 결과가 거짓이기 때문입니다.

A || B (OR):

A가 참(1)이면 B는 아예 평가하지 않습니다. 어차피 결과가 참이기 때문입니다.

i < 10 = 거짓 → ++j는 실행 안 됨 → j = 5 그대로!

삼항 연산자 ( ? : ) — if-else를 한 줄로!

형식: (조건식) ? 참일_때_값 : 거짓일_때_값

삼항 연산자 (한 줄)

result = (age >= 20) ? 1 : 0;

동일한 if-else (여러 줄)

if (age >= 20) result = 1;
else result = 0;

💡 삼항 연산자와 if-else는 완전히 같은 동작을 합니다. 조건이 간단할 때 삼항이 더 간결합니다.

예제
삼항 연산자 (89-2.c)
89-2.c 
#include <stdio.h>
int main()
{
    int i = 13, j = 5, result;
    // "i가 j보다 작니? 맞으면 10, 틀리면 100"
    result = i < j ? 10 : 100;
    // i(13) < j(5) → 거짓 → result = 100
    printf("%d", result);
    return 0;
}
100
13 < 5 는 거짓(0) → 콜론(:) 뒤의 100이 선택됩니다.

Section 6

비트 연산자 (Bitwise)

컴퓨터가 데이터를 저장하는 가장 작은 단위인 0과 1(2진수)을 직접 다루는 연산자입니다. 하드웨어 제어나 성능 최적화에 활용됩니다.

비트 논리 연산자

&AND:둘 다 1이면 1
|OR:하나라도 1이면 1
^XOR:서로 다를 때만 1
~NOT:0↔1 뒤집기

시프트(Shift) 연산자

<< (왼쪽 이동): 2배씩 커집니다.
5 (0101) << 1 = 10 (1010)
>> (오른쪽 이동): 2로 나눕니다.
20 (10100) >> 1 = 10 (01010)

i = 13 (00001101) 과 j = 5 (00000101) 의 비트 연산 시각화

값

j = 5
00000101

※ ~i = -(i+1) = -(13+1) = -14. 부호 비트(맨 앞 비트)가 1이면 음수입니다.

Section 7

형변환 (Type Casting)

C언어에서 정수끼리 나누면 결과도 무조건 정수입니다 (소수점 버림). 소수점이 있는 결과를 얻으려면 형변환(캐스팅)으로 자료형을 임시로 바꿔야 합니다.

❌ 정수 나눗셈 (소수점 사라짐)

int a=5, b=2;

printf("%f", a/b);

→ 출력: 2.000000

정수 나눗셈 먼저 → 2

⚠️ 잘못된 형변환 (여전히 오류)

int a=5, b=2;

printf("%f", (float)(a/b));

→ 출력: 2.000000

a/b=2 먼저, 그후 float

✅ 올바른 형변환

int a=5, b=2;

printf("%f", (float)a/b);

→ 출력: 2.500000

a를 float로 먼저 변환

핵심: (float)(a/b)는 정수 나눗셈이 먼저 일어난 뒤 형변환됩니다. 반드시 (float)a / b 형태로 — 나누기 이전에 형변환해야 소수점이 살아납니다.

Step
형변환 실습: 올바른 vs 잘못된 비교
casting.c 
#include <stdio.h>
int main() {
    int a = 5, b = 2;

    // 1. 정수 나눗셈: 소수점 버림
    printf("int/int        = %d\n", a/b);          // 2

    // 2. 잘못된 형변환: 나눗셈 후 캐스팅 (소수점 이미 사라짐)
    printf("(float)(a/b)   = %f\n", (float)(a/b)); // 2.000000 ← 여전히 틀림!

    // 3. 올바른 형변환: 나누기 전에 캐스팅
    printf("(float)a / b   = %f\n", (float)a / b); // 2.500000 ← 정확!
    printf("(double)a / b  = %lf\n", (double)a / b);// 2.500000 ← 더 정밀

    return 0;
}
int/int = 2
(float)(a/b) = 2.000000
(float)a / b = 2.500000
(double)a / b = 2.500000

Section 8

연산자 응용 예제: 실생활 계산기

지금까지 배운 연산자들을 활용해 실제 생활에 사용할 법한 계산 프로그램을 만들어 봅니다. scanf로 직접 값을 입력받아 결과를 출력합니다.

예제
정기예금 이자 계산기 (92.c)
92.c 
#include <stdio.h>
int main()
{
    int money, duration;
    double rate, result;
    printf("정기예금 금액(원): ");   scanf("%d", &money);
    printf("연 이율(%%): ");         scanf("%lf", &rate);
    printf("예금 기간(년): ");       scanf("%d", &duration);
    // 원리합계 = 원금 + 원금 × 기간 × 이율/100
    result = money + money * duration * 0.01 * rate;
    printf("%d원을 %.1f%%이율로 %d년간 예치 시 원리합계: %.0f원",
        money, rate, duration, result);
    return 0;
}
실행 예시 (100만원, 3%, 2년):
정기예금 금액(원): 1000000↵
연 이율(%): 3.0↵
예금 기간(년): 2↵
1000000원을 3.0%이율로 2년간 예치 시 원리합계: 1060000원
예제
사칙연산 계산기 (93.c) — 형변환 적용
93.c 
#include <stdio.h>
int main()
{
    int a, b;
    printf("두 정수를 입력하세요: ");
    scanf("%d %d", &a, &b);
    printf("%d + %d = %d\n", a, b, a + b);
    printf("%d - %d = %d\n", a, b, a - b);
    printf("%d * %d = %d\n", a, b, a * b);
    // ★ 올바른 형변환: (float)a / b — 나누기 전에 캐스팅!
    printf("%d / %d = %f\n", a, b, (float)a / b);
    printf("%d %% %d = %d\n", a, b, a % b);
    return 0;
}
실행 예시 (5 ÷ 2):
두 정수를 입력하세요: 5 2↵
5 + 2 = 7
5 - 2 = 3
5 * 2 = 10
5 / 2 = 2.500000
5 % 2 = 1
⚠️ 원본 교과서 코드의 (float)(a/b)는 소수점이 사라지는 버그가 있습니다. 올바른 형태는 (float)a / b입니다. (Section 7 참고)
예제
속도 변환: 마일 → km (94-1.c)
94-1.c 
#include <stdio.h>
int main()
{
    int mileSpeed;
    double kmSpeed;
    printf("속도 입력(마일): ");
    scanf("%d", &mileSpeed);
    kmSpeed = mileSpeed * 1.609;
    printf("%d마일 = %.3fkm",
        mileSpeed, kmSpeed);
    return 0;
}
100마일 입력 시: 100마일 = 160.900km

Section 9

연산자 우선순위 종합표

이 우선순위를 모두 외울 필요는 없습니다. 중요한 계산이 있다면 컴퓨터의 기본 규칙에 맡기기보다 괄호( )로 먼저 계산할 부분을 명확히 표시하는 것이 가장 안전합니다. 즉, 반드시 먼저 이루어져야 하는 연산은 괄호로 감싸 두는 습관을 가지는 것이 좋습니다.

우선순위 연산자 종류 결합방향
1 ( ) 괄호 (최우선!) → 왼쪽
2 ++, --, !, ~, (type) 단항 연산자, 형변환 ← 오른쪽
3 *, /, % 곱셈, 나눗셈, 나머지 → 왼쪽
4 +, - 덧셈, 뺄셈 → 왼쪽
5 <<, >> 비트 시프트 → 왼쪽
6 <, <=, >, >= 크기 비교 → 왼쪽
7 ==, != 같다/다르다 → 왼쪽
8~10 &, ^, | 비트 AND/XOR/OR → 왼쪽
11~12 &&, || 논리 AND/OR → 왼쪽
13 ? : 삼항 연산자 ← 오른쪽
14 =, +=, -=, *=, /=, %= 대입 연산자 (최후) ← 오른쪽

우선순위 트레이닝 — 실행 전에 결과를 맞춰보세요!

priority_training.c 
#include <stdio.h>
int main() {
    int a = 5, b = 3, c = 2;

    // 문제 1: 덧셈 vs 곱셈
    printf("%d\n", a + b * c);         // ? ← 곱셈 먼저!

    // 문제 2: 비교 vs 논리
    printf("%d\n", a > b && b > c);    // ? ← 비교 먼저, 그 후 &&

    // 문제 3: 삼항과 대입
    int result = a > b ? a : b;        // ? ← 큰 수 선택
    printf("%d\n", result);

    // 문제 4: 복합 수식 (괄호 없으면?)
    printf("%d\n", a + b * c - a % b); // ? ← *, % 먼저 계산

    return 0;
}
11  (5 + 3×2 = 5+6 = 11)
1   (5>3=1, 3>2=1, 1&&1=1)
5   (5>3이 참 → a=5 선택)
9   (b*c=6, a%b=2 먼저 → 5+6-2=9)

Self-Check

자가점검 — 스스로 답할 수 있나요?

책이나 노트를 보지 않고 답할 수 있으면 이 장의 학습이 완료된 것입니다.

Q1

10 / 3과 10 % 3의 결과는 각각 무엇인가요? 왜 그런가요?

힌트: 정수 나눗셈의 소수점 처리, 나머지 연산

Q2

int a=10; int r = a++; 실행 후 r과 a의 값은?

힌트: 후위 증감 — 대입 먼저, 증가 나중

Q3

C언어에서 비교 연산의 결과값은 무엇인가요? (10 > 5의 결과는?)

힌트: 참=1, 거짓=0

Q4

(float)(5/2)의 출력 결과는 무엇인가요? 올바른 형변환은 어떻게 해야 하나요?

힌트: 나눗셈 실행 순서, (float)5/2

Q5

i=1, j=0일 때 i || ++j의 결과와 j의 값은? (단락 회로)

힌트: OR에서 앞이 참이면 뒤는 평가 안 됨

Q6

result = (a > b) ? a : b;는 어떤 기능을 하나요? if-else로 동일하게 표현해 보세요.

힌트: 삼항 연산자 = 짧은 if-else

정답 확인 (클릭)

A1. 10/3=3 (정수 나눗셈 → 소수점 버림), 10%3=1 (10=3×3+1 → 나머지 1)

A2. r=10, a=11. 후위(a++)는 현재 값(10)을 r에 먼저 대입하고, 그 후 a가 11로 증가합니다.

A3. 결과는 1. C언어는 참=1, 거짓=0으로 표현합니다.

A4. (float)(5/2)는 2.000000. 정수 나눗셈(5/2=2)이 먼저 일어난 후 float로 변환됩니다. 올바른 방법: (float)5/2 → 2.500000.

A5. 결과=1(참), j=0 그대로. OR에서 앞(i=1)이 이미 참이므로 뒤(++j)는 실행되지 않습니다 — 단락 회로 평가.

A6. a와 b 중 큰 값을 result에 저장. if-else 표현: if(a>b) result=a; else result=b;
