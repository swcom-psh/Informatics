Chapter 1. 프로그래밍의 개요

1-2. 프로그래밍의 절차

프로그램을 효율적으로 개발하기 위한 체계적인 절차와 각 단계별 유의사항을 알아봅니다.

Think & Objective

"게임 프로그램을 개발할 때, '베타 테스트'는 과연 어느 과정에서 이루어질까요?"

  • 프로그램을 효율적으로 개발하기 위한 체계적인 절차를 설명할 수 있다.
  • 프로그래밍 절차의 각 단계별로 유의해야 할 사항을 설명할 수 있다.

1. 체계적인 프로그램 개발 절차

컴퓨터에게 어떤 작업을 시키기 위해서는 구체적으로 일을 지시해야 합니다. 효율적이고 정확한 프로그램을 만들기 위해 다음과 같은 4단계의 절차를 거칩니다.

1

문제 분석

해결해야 할 문제가 무엇인지 파악하고, 프로그램이 갖추어야 할 요구사항을 정의합니다.

2

설계 (입출력 설계 / 알고리즘 설계)

문제 해결을 위한 구체적인 방법(알고리즘)을 고안하고, 어떤 데이터를 입력받아 어떻게 출력할지 입출력 설계를 진행합니다.

3

코딩 (프로그램 작성)

주어진 알고리즘에 따라 프로그래밍 언어를 사용하여 명령문을 직접 입력하는 과정입니다.

4

실행 및 평가 (모의 실행)

프로그램이 설계된 대로 잘 작동하는지 가상 데이터를 입력하여 모의 실행을 하고, 오류를 찾아 수정합니다.

2. 오류의 종류와 디버깅(Debugging)

프로그램 작성 도중 발생하는 오류를 찾아 수정하는 과정을 오류 수정(디버깅)이라고 합니다. 오류는 크게 두 가지로 나뉩니다.

문법적 오류

프로그램 작성 중 프로그래밍 언어의 문법과 규칙에 맞지 않게 작성하여, 언어 번역(컴파일) 과정에서 발생하는 오류입니다. 주로 오타 때문에 발생합니다. 

# 문법적 오류 예시 (따옴표 누락)
print(Hello World)
#include <stdio.h>
int main() {
    printf(Hello World);
    return 0;
}

논리적 오류

문법은 맞아서 번역은 되지만, 프로그램의 제어 논리가 잘못되어 뜻하지 않은 결과가 나오는 오류입니다. 모의 실행 단계에서 발견됩니다. 

# 논리적 오류 예시
result = 2 - 3
print(result)
#include <stdio.h>
int main() {
    int result = 2 - 3;
    printf("%d\n", result);
    return 0;
}

3. 빈틈없는 테스트 방법

실행 및 평가 단계에서는 다양한 방법으로 프로그램을 검증해야 합니다.

베타 테스트 (Beta Test)

프로그램 코딩과 입력이 완료된 후, 가상 데이터를 입력하여 프로그램이 설계된 대로 실행되는지 확인하는 [실행 및 평가] 단계에서 이루어지는 테스트입니다.

경계값 검사

경계에서 발생할 수 있는 오류를 집중적으로 검사하는 방법입니다. 시험 대상이 되는 조건의 ① 동일한 값, ② 바로 하위의 값, ③ 바로 상위의 값을 모두 입력하여 시험합니다.

예: 80점 이상 합격일 경우 ➔ 79점(하위), 80점(동일), 81점(상위) 모두 테스트

핵심 요약

코딩 (프로그램 작성)
디버깅 (오류 수정)
문법적 오류 & 논리적 오류
모의 실행 & 베타 테스트

"완벽한 프로그램은 없습니다. 꼼꼼한 모의 실행과 디버깅만이 완성도를 높입니다!"
