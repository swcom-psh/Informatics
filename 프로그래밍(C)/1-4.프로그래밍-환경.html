Chapter 1. 프로그래밍의 개요

1-4. 프로그래밍 환경

우리가 쓴 코드는 어떻게 진짜 프로그램이 될까요? 프로그램의 탄생 과정과 개발을 도와주는 마법의 도구, 통합 개발 환경(IDE)에 대해 알아봅니다.

Think & Objective

"프로그램 작성 시 발생하는 수많은 오류, 어떻게 하면 쉽게 찾고 수정할 수 있을까요?"

  • 프로그램의 개발 과정을 이해하고 통합 개발 환경의 필요성을 설명할 수 있다.
  • 통합 개발 환경을 설치하고 원하는 환경을 설정할 수 있다.

1. 프로그램 개발 과정

우리가 프로그래밍 언어로 작성한 코드(소스 코드)는 컴퓨터가 바로 이해할 수 없습니다. 컴퓨터가 이해할 수 있는 기계어(0과 1)로 번역하고 연결하는 과정이 필수적입니다.

1. 편집 (Edit)

소스 코드 작성

소스 파일

2. 컴파일 (Compile)

기계어로 번역

목적 파일

3. 링크 (Link)

라이브러리 연결

실행 파일

4. 실행 (Execute)

프로그램 구동

결과 확인

필수 개념 짚고 넘어가기!

  • 컴파일 (Compile): 프로그래머가 작성한 소스 코드를 컴퓨터가 이해할 수 있는 기계어(0과 1)로 변환하는 과정입니다. 이때 생성되는 파일이 목적 파일(Object File)입니다.
  • 링크 (Link): 프로그램을 실행하기 위해 서로 연관된 목적 파일들과 미리 만들어진 도구 상자(라이브러리)를 하나로 연결해 최종적인 실행 파일(Executable File)을 만드는 과정입니다.

2. 통합 개발 환경 (IDE)

과거에는 코드를 작성하는 에디터, 번역하는 컴파일러, 연결하는 링커, 오류를 잡는 디버거가 모두 따로 떨어져 있었습니다. 이 모든 것을 하나의 프로그램 안에서 편리하게 사용할 수 있도록 통합해 놓은 소프트웨어를 통합 개발 환경(Integrated Development Environment)이라고 합니다.

에디터 (Editor)

문법 강조(색상 변화) 및 자동 완성 기능으로 빠르고 정확한 코딩 지원

컴파일러 (Compiler)

클릭 한 번으로 문법 오류를 체크하고 빠르게 기계어로 번역

디버거 (Debugger)

오류가 발생한 정확한 위치와 원인을 찾아주어 수정(디버깅)을 도움

디버거의 핵심 명령어

실행 (Go)

프로그램을 중단점(Breakpoint)이 나올 때까지, 혹은 끝까지 한 번에 실행합니다.

한 문장씩 실행 (Step Over)

코드를 한 줄씩 실행합니다. 함수를 만나도 내부로 들어가지 않고 하나의 문장처럼 통과합니다.

함수 내부 진입 (Step Into)

한 줄씩 실행하되, 함수를 만나면 그 함수의 내부 코드로 진입하여 상세히 확인합니다.

중단점 (Breakpoint)

실행 중인 프로그램을 특정 위치에서 일시 정지하도록 마커(빨간 점)를 설정합니다.

핵심 요약

편집 ➔ 컴파일 ➔ 링크 ➔ 실행
통합 개발 환경 (IDE)
디버깅 (Go, Step Over, Step Into, Breakpoint)

"통합 개발 환경(IDE)은 프로그래머의 작업 효율을 극대화시켜 주는 최고의 파트너입니다."
