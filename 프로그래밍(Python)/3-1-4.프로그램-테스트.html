프로그램 테스트: 버그 사냥꾼

작성한 코드가 기획 의도대로, 설계 조건에 맞춰 완벽하게 돌아가는지 검증하고 수정합니다.

구현이 끝났으니 바로 배포할까요?

지난 시간에 우리는 파이썬으로 '성적 계산기' 코드를 완성했습니다. 하지만 100점을 입력했는데 오류가 나거나 프로그램이 꺼져버린다면 어떨까요? 이 문제를 찾아내고 해결하는 과정이 바로 테스트입니다.

단계별 테스트의 종류

소프트웨어 테스트는 작은 단위에서 시작하여 전체 시스템으로 확장하며 꼼꼼하게 진행합니다.

단위 테스트

개별 함수나 모듈이 설계대로 작동하는지 최소 단위로 검사합니다.

구현 예시:
"중간고사 반영 수식"이 정확히 30%를 계산하는지 단일 수식만 확인.

통합 테스트

단위 모듈들을 합쳐서 데이터가 서로 잘 흐르는지 인터페이스를 검사합니다.

구현 예시:
입력받은 값이 계산 함수를 거쳐 출력 화면까지 잘 전달되는지 확인.

시스템 테스트

전체 시스템이 사용자의 요구사항(기획)을 만족하는지 최종 확인합니다.

구현 예시:
실제 학교 성적 산출 기준에 맞춰 소수점 자리수까지 정확한지 최종 점검.

오류 보고서와 디버깅

테스트 중 발견된 예상 결과와 실제 결과의 차이를 기록하고, 원인을 찾아 수정(디버깅)합니다.

입력 데이터 예상 결과 실제 결과 상태
100, 100, 100 입력 100.0점 100.0점 PASS
숫자 대신 'A' 입력 오류 안내 출력 프로그램 강제 종료 FAIL
디버깅(Debug) 계획:

문자 입력 시 발생하는 ValueError를 방지하기 위해, 구현 단계의 코드에 예외 처리(try-except) 구문을 추가하여 보완해야 함!

왜 테스트를 해야 할까요?

  • 품질 보장: 기획했던 모든 기능이 정상적으로 작동하는지 확인합니다.
  • 신뢰성 확보: 사용자가 오류 없이 안심하고 프로그램을 사용할 수 있게 합니다.
  • 비용 절감: 배포 후 발견되는 큰 버그를 미리 방지하여 수정 비용을 줄입니다.
