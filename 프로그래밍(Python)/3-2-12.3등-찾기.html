순위 데이터 정렬: 3등 찾기

이름과 점수가 한 쌍인 다차원 리스트를 정렬하여 특정 순위의 데이터를 추출합니다.

정렬로 순위 찾기

여러 사람의 이름과 점수가 주어졌을 때, 3번째로 높은 점수의 학생을 찾으려면 어떻게 해야 할까요? 파이썬의 sort()와 lambda를 이용하면 단 두 줄로 해결할 수 있습니다. 이번 프로젝트에서는 2차원 리스트 정렬의 핵심 개념을 익힙니다.

Step 1. 프로그램 기획 (문제 정의)

\(n\)명의 친구들의 이름과 정보 성적이 주어집니다. 이 데이터들 중에서 성적이 세 번째로 높은 학생을 찾아 그 이름을 출력하는 프로그램을 기획합니다.

2등
이민수 (95)
1등
김철수 (98)
3등
박영희 (92)

데이터 예시: [ ['김철수', 98], ['이민수', 95], ['박영희', 92], ... ]

입력 조건

첫 줄에 인원 수 n, 이후 n줄에 걸쳐 이름과 점수가 공백으로 구분되어 입력됩니다.

출력 조건

점수 기준 3등(세 번째로 높은 점수)인 학생의 이름을 출력합니다.

Step 2. 프로그램 설계 (알고리즘)

1. 데이터 구조화

이름(String)과 점수(Int)를 하나의 묶음으로 처리하기 위해 2차원 리스트를 활용합니다.
예: ['김철수', 98]

2. 정렬 기준 설정

리스트의 인덱스 1(점수)을 기준으로 내림차순 정렬을 수행해야 합니다.

핵심 아이디어: 파이썬의 sort() 함수와 lambda 식을 이용하면 다차원 리스트를 특정 열 기준으로 쉽게 정렬할 수 있습니다.

lambda 문법 완전 분해

lambda는 이름 없는 간단한 함수를 한 줄로 만드는 문법입니다. 각 부분이 무엇을 의미하는지 확인하세요.

lambda키워드
x매개변수
(원소 하나)
:구분자
x[1]반환값
(점수 추출)
# lambda를 일반 함수로 풀어쓰면:
def get_score(x):
    return x[1] # x = ['김철수', 98] → x[1] = 98

# lambda 한 줄로 압축:
key = lambda x: x[1] # 완전히 동일한 동작!
sort() vs sorted() 차이
students.sort(...): 원본 리스트를 직접 수정 (in-place), 반환값 없음 (None)
sorted(students, ...): 원본 유지, 새 리스트 반환 → 원본이 필요하면 sorted 사용

Step 3. 프로그램 구현 (Python)

⚠️ n < 3 입력 시 IndexError

n이 1 또는 2이면 students[2]에 접근할 때 IndexError: list index out of range가 발생합니다. 문제 조건상 n ≥ 3이 보장되는지 확인하세요. 방어 처리가 필요하면 아래 심화 코드를 참고하세요.

Python Solution
# 1. n명의 정보 입력 받기
n = int(input())
students = []

for _ in range(n):
    name, score = input().split()
    students.append([name, int(score)])
    # ✅ int(score) 필수 — 문자열로 받으면 '9' > '10' 처럼 잘못 비교됨

# 2. 점수(index 1) 기준 내림차순 정렬
students.sort(key=lambda x: x[1], reverse=True)
# key=lambda x: x[1] → 각 원소의 두 번째 값(점수)을 기준으로
# reverse=True → 내림차순 (높은 점수가 앞으로)

# 3. 3등(index 2) 학생의 이름 출력
# 정렬 후: [0]=1등, [1]=2등, [2]=3등
print(students[2][0])
💡 인덱스와 순위의 관계

파이썬 리스트 인덱스는 0부터 시작합니다. 정렬 후 순위와 인덱스의 관계:

students[0] # 1등 (가장 높은 점수)
students[1] # 2등
students[2] # 3등 ← 우리가 찾는 것
students[2][0] # 3등의 이름 (첫 번째 원소)
students[2][1] # 3등의 점수 (두 번째 원소)
⚠️ 동점자 처리 주의

1등과 2등이 동점(예: 98, 98, 92)이라면 students[2]는 여전히 3번째 위치의 학생을 반환합니다. 문제에서 "3번째로 높은 점수" 또는 "3번째 순위"를 구분해야 합니다.

# 예: 98, 98, 92 → 정렬 후 순서: [98, 98, 92]
# students[2] = 92점 학생 (3번째 위치) ← 이 코드의 동작
# "3번째로 다른 점수" = 92 → 동일 결과
# 예: 98, 98, 98 → students[2] = 98점 학생 (공동 1등)
# 문제 조건에서 동점이 없다고 가정하는 경우가 많습니다.

코드 실행 추적 (Trace) — 4명 입력 예시

입력: n=4, 김철수 98 / 이민수 95 / 박영희 92 / 최지훈 88

① 입력 후 students 리스트:

[['김철수', 98], ['이민수', 95], ['박영희', 92], ['최지훈', 88]]

② sort(key=lambda x: x[1], reverse=True) 적용 후:

인덱스 이름 점수 순위
[0] 김철수 98 🥇 1등
[1] 이민수 95 🥈 2등
[2] 박영희 92 🥉 3등 ← 정답
[3] 최지훈 88 4등

print(students[2][0]) → 박영희

students.append([name, score]) # ❌ score = "98" (문자열)
