알고리즘의 이해

"Logic is the beginning of wisdom."

생각의 지도, 알고리즘

어떤 문제를 해결하기 위해 거쳐야 하는 명확한 절차나 방법입니다. 좋은 알고리즘은 복잡한 문제도 빠르고 정확하게 풀어냅니다.

입력/출력

0개 이상의 입력과 1개 이상의 결과가 있어야 합니다.

유한성

반드시 실행 후에 종료되어야 합니다. 우주가 끝날 때까지 계산해도 결과를 도출할 수 없다면 그건 잘못된 알고리즘입니다.

명확성

모든 단계는 실행가능하도록 구체적이고 명확해야 합니다.

A

의사 코드 (Pseudo Code)

특정 언어의 문법은 아니지만, 사람이 이해하기 쉬운 말로 논리를 표현합니다.

B

순서도 (Flowchart)

정해진 기호를 사용하여 처리 과정의 흐름을 시각적으로 나타냅니다.

C

프로그래밍 언어 (Programming Language)

특정 프로그래밍 언어에 맞는 문법과 규칙에 따라 알고리즘을 표현하고 바로 실행할 수 있습니다.

좋은 알고리즘은 누구나 같은 결과를 얻을 수 있도록 객관적이고 명확하게 기술되어야 합니다. 두 개의 자연수 a, b의 최대 공약수를 출력하는 알고리즘을 표현해 봅시다.
