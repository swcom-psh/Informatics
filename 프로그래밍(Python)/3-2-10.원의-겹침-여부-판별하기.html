원의 겹침 여부 판별하기

두 원의 좌표와 반지름을 이용하여 논리적으로 위치 관계를 계산하는 프로그램

두 원의 위치 관계를 코드로!

두 원의 위치 관계는 두 중심 사이의 거리(d)와 두 반지름의 합/차를 비교하여 결정됩니다. 이번 프로젝트에서는 이 수학적 조건을 파이썬 조건문으로 변환하는 연습을 합니다. 특히 5가지 경우를 정확히 처리하는 것이 핵심입니다.

Step 1. 프로그램 기획 (문제 정의)

두 원의 중심 좌표 \((x_1, y_1),\ (x_2, y_2)\)와 각 원의 반지름 \(r_1,\ r_2\)를 입력받아, 두 원이 어떤 상태에 있는지 판별하는 프로그램을 개발합니다.

두 원의 위치 관계 — 5가지 경우

외부 (분리)
d > r1+r2
만나지 않는다
외접
d = r1+r2
한 점에서 만난다
두 점 교차
|r1-r2| < d < r1+r2
두 점에서 만난다
내접
d = |r1-r2|
한 점에서 만난다
내부 포함
d < |r1-r2|
만나지 않는다

입력 조건

두 줄로 입력됩니다.
1행: x1 y1 r1
2행: x2 y2 r2

출력 조건

위치 관계에 따라 3가지 중 하나를 출력합니다.
"두 점에서 만난다"
"한 점에서 만난다"
"만나지 않는다"

Step 2. 프로그램 설계 (알고리즘)

핵심 알고리즘은 '두 점 사이의 거리 공식'과 '반지름의 합/차'를 비교하는 것입니다.

두 중심 사이의 거리 \(d\) = \(\sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}\)
🔴 원본 오류 수정 — 3케이스 → 5케이스로 완전 교정

원본의 판별 로직은 3가지 경우만 처리해 내접(d=|r1-r2|)과 내부 포함(d<|r1-r2|) 케이스를 모두 "두 점에서 만난다"로 잘못 출력했습니다. 올바른 판별에는 5가지 조건이 필요합니다.

완전한 판별 로직 (5케이스)

# 조건 기하학적 의미 출력
1 \(d > r_1 + r_2\) 외부에서 분리 (외부 불만남) 만나지 않는다
2 \(d = r_1 + r_2\) 외접 (바깥에서 한 점 접촉) 한 점에서 만난다
3 \(|r_1 - r_2| < d < r_1 + r_2\) 두 점에서 교차 두 점에서 만난다
4 \(d = |r_1 - r_2|\) 내접 (안쪽에서 한 점 접촉) 한 점에서 만난다
5 \(d < |r_1 - r_2|\) 한 원이 다른 원 내부에 포함 만나지 않는다
🔴 부동소수점 비교 오류 — math.sqrt 사용 시 위험

d = math.sqrt(...)의 결과는 float입니다. d == r1 + r2처럼 float를 정수와 `==`로 비교하면 부동소수점 오차로 외접/내접 판별이 실패할 수 있습니다.

# ❌ 위험한 비교 (float 오차 발생 가능)
d = math.sqrt(5)  # 실제: 2.2360679...
d == 2          # False ← 수학적으로도 False지만, 정수 좌표에서 오차 위험

# ✅ 안전한 비교 (제곱으로 비교 — 정수 연산만 사용)
d_sq = (x2-x1)**2 + (y2-y1)**2
d_sq == (r1 + r2)**2   # 정수 == 정수 → 완전 정확

Step 3. 프로그램 구현 (Python)

조건문과 제곱 비교를 활용하여 부동소수점 오차 없이 구현합니다.

🔴 원본 코드 — else 분기 논리 오류

원본의 else: 분기는 d < r1+r2인 모든 경우를 "두 점에서 만난다"로 출력합니다. 이 범위 안에는 내접과 내부 포함도 포함되어 있어 완전히 잘못된 결과를 냅니다.

# ❌ 원본 코드 (오류 — 3케이스만 처리)
if d > r1 + r2:
    print("만나지 않는다")
elif d == r1 + r2: # ⚠️ float == int 비교 위험
    print("한 점에서 만난다")
else: # ❌ 내접·내부포함도 여기로 빠짐!
    print("두 점에서 만난다")
✅ 수정 완료 — 5케이스 완전 처리 + 제곱 비교

math.sqrt를 제거하고 제곱 비교를 사용해 정수 연산만으로 정확히 판별합니다.

Python Solution (수정 완료)
# 1. 두 원의 정보 입력
x1, y1, r1 = map(int, input().split())
x2, y2, r2 = map(int, input().split())

# 2. 제곱 거리 계산 (sqrt 미사용 → 부동소수점 오차 방지)
d_sq = (x2 - x1)**2 + (y2 - y1)**2

# 3. 반지름 합/차의 제곱 (정수 비교를 위해)
sum_sq = (r1 + r2)**2     # 외접/분리 경계
diff_sq = (abs(r1 - r2))**2 # 내접/포함 경계

# 4. 5케이스 완전 판별
if d_sq > sum_sq:
    print("만나지 않는다")  # 케이스 1: 외부 분리
elif d_sq == sum_sq:
    print("한 점에서 만난다")  # 케이스 2: 외접
elif d_sq > diff_sq:
    print("두 점에서 만난다")  # 케이스 3: 두 점 교차
elif d_sq == diff_sq:
    print("한 점에서 만난다")  # 케이스 4: 내접
else:
    print("만나지 않는다")   # 케이스 5: 내부 포함 (d_sq < diff_sq)
왜 math.sqrt를 쓰지 않나요?

비교 대상이 모두 정수(r1, r2)이므로 양변을 제곱하면 제곱근 없이 정수만으로 비교할 수 있습니다.
\(d > r_1 + r_2\)  ↔  \(d^2 > (r_1 + r_2)^2\) (d ≥ 0 이므로 동치)

코드 실행 추적 (Trace) — 5가지 대표 입력

각 케이스별 핵심 계산값을 단계별로 추적합니다.

케이스 입력 (x1 y1 r1 / x2 y2 r2) d² sum² diff² 조건 출력
외부 분리 0 0 3 / 10 0 3 100 36 0 100 > 36 만나지 않는다
외접 0 0 3 / 6 0 3 36 36 0 36 == 36 한 점에서 만난다
두 점 교차 0 0 3 / 4 0 3 16 36 0 0 < 16 < 36 두 점에서 만난다
내접 0 0 5 / 2 0 3 4 64 4 4 == 4 한 점에서 만난다
내부 포함 0 0 5 / 1 0 2 1 49 9 1 < 9 만나지 않는다
