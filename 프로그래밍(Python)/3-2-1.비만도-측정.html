비만도 측정 프로그램 개발

기획부터 테스트까지, SDLC 과정을 따라 나만의 비만도 측정기를 완성해 봅시다.

왜 비만도 측정인가요?

프로그래밍 실력을 키우는 가장 좋은 방법은 실생활의 문제를 코드로 해결해 보는 것입니다. 이번 프로젝트에서는 키와 몸무게 데이터를 활용해 복합적인 조건문을 설계하는 연습을 합니다.

Step 1. 프로그램 기획 (문제 분석)

사용자가 키와 몸무게를 입력하면, 표준 몸무게를 기준으로 비만도를 계산하고 상태를 알려주어야 합니다.

입력 요구 사항

사용자의 키(cm)와 몸무게(kg)를 공백을 두고 입력받습니다.

처리 요구 사항

키에 따른 표준 몸무게를 구하고, 비만도(%) 수식을 적용합니다.

출력 요구 사항

비만도 수치에 따라 정상, 과체중, 비만 중 하나를 출력합니다.

Step 2. 프로그램 설계 (알고리즘)

컴퓨터가 계산할 수 있도록 단계를 나누어 설계해 봅시다. (표준 몸무게 계산 방식)

1) 키에 따른 표준 몸무게(SW) 계산법
H < 150cm : SW = H − 100
150 ≤ H < 160cm : SW = (H − 150) / 2 + 50
H ≥ 160cm : SW = (H − 100) × 0.9
2) 비만도 수식
비만도(%) = (실제 몸무게 − 표준 몸무게) × 100 / 표준 몸무게
설계 시 주의!

비만도가 10 이하면 '정상', 10 초과 20 이하면 '과체중', 20 초과면 '비만'으로 판정합니다. 이때 조건의 경계값(10, 20)을 포함할지 말지 설계서에 명확히 적어야 합니다.

비만도 판정 기준 한눈에 보기

😟 저체중
비만도 < 0
(실제체중 < 표준체중)
😊 정상
0 ≤ 비만도 ≤ 10
⚠️ 과체중
10 < 비만도 ≤ 20
🚨 비만
비만도 > 20
🔍 학습 포인트 — 저체중(음수 비만도)

비만도 수식에서 실제 몸무게가 표준 몸무게보다 적으면 음수가 나옵니다.
예: 키 170cm, 몸무게 55kg → sw = (170-100)×0.9 = 63, 비만도 = (55-63)×100/63 ≈ -12.7%

이 경우 obesity < 0이 되므로 이론상 저체중이지만, 이번 프로그램에서는 obesity <= 10 조건을 먼저 평가하기 때문에 "정상"으로 출력됩니다.
👉 이것은 의도적 단순화입니다. 실제 건강 앱이라면 obesity < 0 분기를 별도로 추가해야 합니다.

💡 확장 과제: 저체중 분기를 추가해 보세요! if obesity < 0: print("저체중")을 가장 앞에 배치하면 됩니다.

Step 3. 프로그램 구현 (Source Code)

설계한 알고리즘을 파이썬 문법으로 옮겨봅시다.

Python Solution
# 1. 입력 받기 (문자열을 실수형으로 변환)
height, weight = map(float, input().split())

# 2. 키에 따른 표준 몸무게(sw) 계산
if height < 150:
    sw = height - 100
elif height < 160:
    sw = (height - 150) / 2 + 50
else:
    sw = (height - 100) * 0.9

# 3. 비만도 계산
obesity = (weight - sw) * 100 / sw

# 4. 결과 판정 및 출력
if obesity <= 10:
    print("정상")
elif obesity <= 20:
    print("과체중")
else:
    print("비만")
✅ 확장 버전 (저체중 분기 포함)

아래는 저체중까지 구분하는 완전한 버전입니다. 기본 버전을 완전히 이해한 뒤 도전해 보세요!

Python Extended
height, weight = map(float, input().split())

if height < 150:
    sw = height - 100
elif height < 160:
    sw = (height - 150) / 2 + 50
else:
    sw = (height - 100) * 0.9

obesity = (weight - sw) * 100 / sw

# ✅ 저체중 분기를 가장 먼저 검사!
if obesity < 0:
    print("저체중")
elif obesity <= 10:
    print("정상")
elif obesity <= 20:
    print("과체중")
else:
    print("비만")
친절한 문법 설명: map(float, ...)

input().split()은 데이터를 공백으로 나누어 가져오지만, 기본적으로 문자열(str) 상태입니다. 이를 한꺼번에 실수형(float)으로 바꾸기 위해 map() 함수를 사용하면 코드가 훨씬 간결해집니다!

한 줄씩 쓴다면:

data = input().split()  # ['170', '75'] ← 문자열 리스트
height = float(data[0])  # 첫 번째 값만 float으로 변환
weight = float(data[1])  # 두 번째 값만 float으로 변환

map()은 이 세 줄을 한 줄로 압축해주는 고마운 함수입니다! 😊

코드 실행 추적 (Trace) — 입력: 170 80

코드가 실제로 어떻게 실행되는지 한 단계씩 따라가 봅시다.

단계 코드 계산 과정 결과값
① height, weight = map(...) 입력 문자열 "170 80" → float 변환 height = 170.0, weight = 80.0
② if height < 150 → False
elif height < 160 → False
else: 실행		 170 ≥ 160 이므로 else 분기 sw = (170 - 100) × 0.9 = 63.0
③ obesity = (weight - sw) * 100 / sw (80 - 63) × 100 ÷ 63 obesity ≈ 26.98%
④ if obesity <= 10 → False
elif obesity <= 20 → False
else: 실행		 26.98 > 20 이므로 else 분기 출력: "비만"

Step 4. 디버깅 및 테스트

프로그램이 다양한 입력값에 대해 정확한 결과를 내는지 확인합시다. 특히 경계값(boundary value) 테스트가 중요합니다!

학생들이 가장 많이 하는 실수!
  • 연산 순서 착각: (weight - sw) * 100 / sw에서 괄호를 빠뜨리면 파이썬은 weight - sw * 100 / sw로 계산합니다. 반드시 괄호를 확인하세요.
  • 들여쓰기 오류: if-elif-else의 시작 세로줄이 맞지 않으면 파이썬은 실행되지 않습니다.
  • 비교 연산자: <=(작거나 같다)와 <(작다)의 차이를 명확히 구분했나요?
  • float vs int: map(int, ...)으로 받으면 170.5cm 같은 소수점 키를 처리하지 못합니다. 항상 float으로 받으세요.

아래 입력값으로 직접 테스트하며 결과를 확인해 보세요.

저체중 (확장 버전)		 음수 비만도 ← 저체중 판별
왜 경계값(10, 20)을 직접 계산해서 넣어봐야 하나요?

obesity <= 10과 obesity < 10은 딱 10일 때만 결과가 달라집니다. 테스트 케이스 #2, #3처럼 경계값 바로 아래/위 숫자를 넣어 보면 자신의 코드가 조건을 올바르게 처리하는지 즉시 확인할 수 있습니다.
