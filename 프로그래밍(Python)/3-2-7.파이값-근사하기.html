확률로 푸는 수학: 파이(π)값 근사하기

무작위 좌푯값을 이용하여 원주율을 계산하는 시뮬레이션 프로그램을 제작합니다.

다트 던지기로 π 찾기?

한 변의 길이가 1인 정사각형 안에 무작위로 점을 찍는다고 상상해 보세요. 그중 반지름이 1인 사분원(부채꼴) 안에 들어가는 점의 비율은 얼마나 될까요? 이 기하학적 확률을 이용하면 신비로운 숫자 π를 프로그래밍으로 찾아낼 수 있습니다.

Step 1. 프로그램 기획 (요구 사항)

무작위로 생성된 \(n\)개의 좌푯값이 주어졌을 때, 공식에 따라 \(\pi\)의 근삿값을 계산합니다.

[단위 정사각형 안의 사분원 — 몬테카를로 시뮬레이션]

(0,0)
(1,0)
(0,1)
(1,1)

사분원 내부 (x²+y²≤1)    사분원 외부 (x²+y²>1)

⚠️ 입력 방식 명확화 — 원본에 input() 없음

원본 코드는 좌표를 하드코딩했습니다. 실제 온라인 채점 시스템에서는 아래와 같이 입력을 받습니다. 두 방식을 구분하여 학습하세요.

# 방법 ①: 하드코딩 (학습/테스트용)
points = [[0.1, 0.2], [0.9, 0.8], ...]

# 방법 ②: 표준 입력 (채점 시스템용)
n = int(input())                   # 점의 개수 입력
points = []
for _ in range(n):
    x, y = map(float, input().split())
    points.append([x, y])

# 방법 ③: random 모듈 (확장 버전 — Step 4 참고)

입력 조건

전체 점의 개수 \(n\)과 각 점의 \((x, y)\) 좌표.
좌표 범위: 0.0 ≤ x, y ≤ 1.0

출력 결과

계산된 \(\pi\)의 근삿값을 출력합니다.
점이 많을수록 3.14159...에 가까워집니다.

Step 2. 프로그램 설계 (핵심 공식)

몬테카를로 알고리즘의 핵심 로직을 설계합니다.

원 안에 있는지 판별하는 법

점 \((x, y)\)와 원점 \((0, 0)\) 사이의 거리가 반지름 \(1\)보다 작거나 같으면 사분원 내부에 있습니다.

$$x^2 + y^2 \leq 1$$

※ 거리 = √(x²+y²) 이므로, 제곱근을 계산하지 않고 제곱 상태에서 비교하면 더 효율적입니다.

π 근사 공식

정사각형 면적(1×1=1)에 대한 사분원 면적(π×1²/4=π/4)의 비율이 점의 비율과 같습니다. 따라서:

$$\pi \approx 4 \times \frac{\text{원 안의 점 개수}}{\text{전체 점의 개수}}$$

💡 왜 4를 곱할까요?

정사각형(넓이=1) 안에서 사분원이 차지하는 비율 = π/4
따라서: 원 안의 점 / 전체 점 ≈ π/4
양변에 4를 곱하면: π ≈ 4 × (원 안의 점 / 전체 점)

좌표 유효 범위: [0, 1] × [0, 1]

이 프로그램은 모든 좌표가 0.0 ≤ x ≤ 1.0, 0.0 ≤ y ≤ 1.0 범위 안에 있다고 가정합니다.
음수나 1을 초과하는 좌표를 입력하면 수학 모델 자체가 무너져 π 근사값이 크게 틀립니다. 입력 범위를 항상 확인하세요.

Step 3. 프로그램 구현 (Source Code)

Python — 기본 버전 (하드코딩)
# 1. 테스트용 좌표 데이터 (2차원 리스트)
# 각 원소 [x, y]는 0.0~1.0 범위의 실수
points = [[0.1, 0.2], [0.9, 0.8], [0.3, 0.4], [0.5, 0.5]]
n = len(points)                  # n = 4
in_count = 0                  # 원 안에 들어간 점의 개수

# 2. 각 좌표를 순회하며 사분원 안에 있는지 검사
for p in points:
    x = p[0]     # p[0]: x 좌표
    y = p[1]     # p[1]: y 좌표
    if x**2 + y**2 <= 1: # 제곱 합이 1 이하 → 원 내부
        in_count += 1

# 3. 파이값 계산 및 출력
pi_approx = 4 * (in_count / n)  # 결과: 4 * (3/4) = 3.0
print("파이 근삿값:", pi_approx)
2차원 리스트 다루기

points[i][0]은 \(i\)번째 점의 x좌표, points[i][1]은 y좌표를 의미합니다. for p in points로 순회할 때 p는 [x, y] 리스트 하나를 받습니다.

⚠️ n = 0 → ZeroDivisionError

좌표 리스트가 비어 있으면 in_count / n에서 ZeroDivisionError: division by zero가 발생합니다. 입력 전에 반드시 n > 0임을 확인하세요.

# 심화 과제: 방어 처리 추가
if n == 0:
    print("오류: 점이 1개 이상 필요합니다.")
else:
    pi_approx = 4 * (in_count / n)
    print(pi_approx)

코드 실행 추적 (Trace) — 4개 점 하드코딩 테스트

각 점의 판별 과정을 단계별로 추적합니다.

반복 p x y x²+y² ≤1? in_count
1 [0.1, 0.2] 0.1 0.2 0.01+0.04=0.05 ✅ 내부 1
2 [0.9, 0.8] 0.9 0.8 0.81+0.64=1.45 ❌ 외부 1
3 [0.3, 0.4] 0.3 0.4 0.09+0.16=0.25 ✅ 내부 2
4 [0.5, 0.5] 0.5 0.5 0.25+0.25=0.50 ✅ 내부 3

n=4, in_count=3
pi_approx = 4 × (3/4) = 3.0
# 실제 π = 3.14159... 점이 4개라 오차 큼 → 점이 많을수록 정확!

Step 4. 프로그램 테스트 및 고도화

점의 개수(n)에 따른 π 근삿값 수렴 (실험적 예시)

점이 많아질수록 실제 π=3.14159...에 수렴합니다. 아래는 random.random()으로 10회 반복 실험 평균값입니다.

결과의 정확성

점의 개수가 많아질수록(예: 10,000개) \(\pi\) 값인 3.14159...에 점점 더 가까워집니다. 이것이 큰 수의 법칙입니다.

랜덤 모듈 활용

직접 좌표를 입력하는 대신 random.random() 함수를 써서 컴퓨터가 스스로 점을 찍게 해보세요. 아래 확장 코드를 참고하세요.

✅ 확장 버전 — random 모듈로 자동 시뮬레이션

직접 좌표를 입력하지 않고, 컴퓨터가 난수를 생성하여 시뮬레이션을 수행합니다.

Python — 확장 버전 (random 모듈)
import random

n = int(input("점의 개수를 입력하세요: "))
in_count = 0

for _ in range(n):
    # random.random()은 0.0 이상 1.0 미만의 난수를 반환
    x = random.random()
    y = random.random()
    if x**2 + y**2 <= 1:
        in_count += 1

pi_approx = 4 * in_count / n
print(f"n={n:,}개 → π ≈ {pi_approx:.5f}")
생각해보기

왜 하필 4를 곱할까요? 정사각형(넓이=1) 안에서 사분원의 넓이는 \(\pi r^2 / 4 = \pi/4\)입니다. 따라서 점 비율은 \(\pi/4\)에 가까워지고, 4를 곱하면 \(\pi\)를 얻습니다.
