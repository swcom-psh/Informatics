이달은 며칠까지 있을까?

리스트와 조건문을 활용하여 특정 연도와 월의 마지막 날을 계산해 봅시다.

프로젝트 배경

매달 말일은 30일이기도 하고 31일이기도 합니다. 특히 2월은 윤년 여부에 따라 28일 또는 29일이 됩니다. 규칙적인 데이터를 효율적으로 관리하는 리스트(List) 기능을 사용하여 이 문제를 해결해 봅시다.

Step 1. 프로그램 기획 (요구 사항)

사용자로부터 연도와 월을 입력받아, 해당 월의 마지막 날짜를 출력합니다.

입력 예시

공백을 구분자로 연도와 월이 입력됩니다.
예: 2024 2

출력 예시

해당 월의 마지막 날짜 숫자를 출력합니다.
예: 29

2) 윤년(Leap Year) 판별 조건식

2월의 말일을 결정하는 윤년의 규칙은 다음과 같습니다.

  • 연도가 400의 배수이면 → 윤년
  • 연도가 4의 배수이면서 100의 배수가 아니면 → 윤년

\((year \% 400 == 0) \text{ or } (year \% 4 == 0 \text{ and } year \% 100 \neq 0)\)

윤년 예시 (2000~2104)

2100100배수 ❌
2200100배수 ❌
2300100배수 ❌
2400400배수 ✅
20244배수 ✅
20284배수 ✅
2023평년 ❌
⚠️ 논리 연산자 우선순위 — 괄호가 없으면 결과가 달라집니다

파이썬에서 and는 or보다 먼저 계산됩니다. 괄호 없이 쓰면 의도하지 않은 결과가 나올 수 있습니다.

# 올바른 코드 (괄호 O)
(year % 400 == 0) or (year % 4 == 0 and year % 100 != 0)

# 위험한 코드 (괄호 X — and가 먼저 묶임)
year % 400 == 0 or year % 4 == 0 and year % 100 != 0
# ↑ 실제로는 (400==0) or ((4==0) and (100!=0)) 로 계산되어 결과가 같을 수도 있지만,
# 복잡한 조건에서는 반드시 괄호로 명확히 표현하는 것이 좋습니다.

Step 3. 프로그램 구현 (Source Code)

Python Solution
# 1. 입력 받기 (연도와 월)
year, month = map(int, input().split())

# 2. 각 달의 마지막 날 리스트 (1월=인덱스0 ~ 12월=인덱스11)
days = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]

# 3. 윤년인 경우 2월(인덱스 1)의 날짜를 29로 수정
if (year % 400 == 0) or (year % 4 == 0 and year % 100 != 0):
    days[1] = 29

# 4. 결과 출력 (month-1 = 리스트 인덱스)
# 예: month=2 → days[1] (2월의 마지막 날)
print(days[month - 1])
친절한 문법 설명: 인덱스(Index)의 함정

우리는 보통 1월을 '첫 번째 달'이라고 부르지만, 파이썬 리스트의 첫 번째 칸 번호는 0입니다. 따라서 사용자가 입력한 month에서 1을 빼주어야 정확한 데이터를 가져올 수 있습니다.

# month=1 (1월) → days[1-1] = days[0] = 31 ✅
# month=2 (2월) → days[2-1] = days[1] = 28 (평년) ✅
# month=12(12월) → days[12-1] = days[11] = 31 ✅

코드 실행 추적 (Trace) — 입력: 2024 2 (윤년 2월)

코드가 실제로 어떻게 실행되는지 한 단계씩 따라가 봅시다.

단계 코드 계산 과정 결과값
① year, month = map(int, input().split()) "2024 2" → int 변환 year=2024, month=2
② days = [31, 28, 31, ...] 12개월 말일 리스트 초기화 days[1] = 28 (초기값)
③ 2024 % 400 == 0 ? 2024 % 400 = 24 ≠ 0 → False 첫 번째 조건: False
④ 2024 % 4 == 0 and 2024 % 100 != 0 ? 2024%4=0 ✅, 2024%100=24≠0 ✅ → True 두 번째 조건: True
⑤ False or True → if 블록 실행 윤년 확정 → days[1] = 29 days[1] = 29 (수정)
⑥ print(days[month - 1]) days[2-1] = days[1] = 29 출력: 29

Step 4. 테스트 및 디버깅

다양한 입력으로 프로그램을 검증합니다. 특히 2월 + 윤년/평년 경계값과 2월이 아닌 달도 반드시 테스트해야 합니다.

테스트 케이스 표 (경계값 포함)

# 입력 year month 윤년? 기대 출력 테스트 목적
1 2024 2 2024 2 ✅ (4배수) 29 기본 일반 윤년 2월
2 2023 2 2023 2 ❌ 28 기본 평년 2월
3 2000 2 2000 2 ✅ (400배수) 29 경계 400배수 윤년
4 2100 2 2100 2 ❌ (100배수) 28 경계 100배수 평년 (가장 많이 틀림!)
5 2024 1 2024 1 — 31 기본 2월 외 31일 달
6 2024 4 2024 4 — 30 기본 2월 외 30일 달
7 2024 12 2024 12 — 31 기본 마지막 달 (인덱스 경계)
8 2024 13 2024 13 — IndexError 오류 유효하지 않은 월 입력
⚠️ month 범위 초과 → IndexError

days 리스트는 인덱스 0~11(총 12개)만 존재합니다. month=13을 입력하면 days[12]에 접근하게 되어 IndexError: list index out of range가 발생합니다.
마찬가지로 month=0이면 days[-1] = 31이 출력되어(12월 값) 오답이 됩니다.

# 입력 유효성 검사를 추가하면 더 완성도 높은 프로그램이 됩니다 (심화 과제)
if 1 <= month <= 12:
    print(days[month - 1])
else:
    print("오류: 월은 1~12 사이여야 합니다.")
학생들이 자주 하는 실수!
  • 논리 연산자 우선순위: and가 or보다 먼저 계산됩니다. 윤년 조건은 괄호를 반드시 사용하여 의도를 명확히 표현하세요. (Step 2 참고)
  • 리스트 수정 순서: days[1] = 29는 반드시 print() 앞에 실행되어야 합니다. 아래 두 코드의 차이를 확인하세요.
# ❌ 잘못된 순서 — print 이후에 리스트를 수정해도 이미 출력된 후!
print(days[month - 1])  # 아직 28 출력
if ...: days[1] = 29       # 너무 늦음

# ✅ 올바른 순서 — 리스트 수정 후 출력
if ...: days[1] = 29
print(days[month - 1])  # 29 정상 출력
  • 인덱스 오프셋 누락: days[month]으로 쓰면 1월이 인덱스 1(= 2월 값 28)이 됩니다. 반드시 days[month - 1]로 작성하세요.
