복면산(覆面算) 문제 해결

글자로 숨겨진 숫자를 논리적 추론과 반복문으로 찾아내는 퍼즐 프로그램

복면산이란?

복면산(覆面算)이란 수식의 각 글자가 0~9 중 하나의 숫자를 나타내는 수학 퍼즐입니다. 같은 글자는 반드시 같은 숫자를, 서로 다른 글자는 서로 다른 숫자를 의미합니다. 이번 프로젝트에서는 컴퓨터의 빠른 연산 능력을 활용한 브루트 포스(완전 탐색) 기법으로 이 문제를 해결합니다.

Step 1. 프로그램 기획 (문제 정의)

복면산이란 각 문자가 서로 다른 숫자를 나타내는 수학 퍼즐입니다. 같은 문자는 같은 숫자를 의미하며, 수의 맨 앞자리(최고차항)는 0이 될 수 없다는 규칙이 있습니다.

  A B
+ B A
─────
  C C

위 문제에서 A, B, C가 각각 어떤 숫자일 때 식이 성립하는지 찾아내는 프로그램을 개발합니다.

제약 조건

  • A, B, C는 모두 서로 다른 숫자
  • A ≠ 0 (AB의 첫 자리)
  • B ≠ 0 (BA의 첫 자리)
  • C ≠ 0 (CC의 첫 자리)

수식 관계

AB + BA = CC
즉, 두 자리수 AB와 그 숫자를 뒤집은 BA의 합이 CC(같은 숫자 두 개)여야 합니다.

Step 2. 프로그램 설계 (알고리즘)

핵심 로직: 브루트 포스 (Brute Force)

컴퓨터의 빠른 연산 능력을 활용하여 가능한 모든 숫자 조합(1~9)을 대입해보고 조건에 맞는 답을 찾습니다.

수식 수학적 유도 — 왜 A+B=C인가요?

코드의 조건식이 어디서 나왔는지 단계별로 유도해 봅니다.

AB + BA = (10A+B) + (10B+A) = 11A + 11B = 11(A+B)
CC = 10C + C = 11C
AB + BA = CC → 11(A+B) = 11C → ∴ A + B = C

💡 즉 코드의 조건식 (a*10+b)+(b*10+a) == (c*10+c)는 결국 a+b == c와 동일합니다.

브루트 포스 탐색 규모

A: 1~9 (9가지) × B: 1~9 (9가지) × C: 1~9 (9가지) = 최대 729번 탐색합니다. 컴퓨터로는 순식간에 끝나는 양입니다. continue로 중복 조합을 건너뛰면 실제 조건 확인 횟수는 더 줄어듭니다.

1. 범위 설정

A, B는 두 자리 수의 첫째 자리이므로 1~9.
C도 CC의 첫째 자리이므로 1~9.

2. 중복 제약

A, B, C는 서로 다른 숫자여야 합니다.
A≠B, B≠C, A≠C

3. 수식 검증

(10A+B)+(10B+A) == (10C+C)
즉 11(A+B) == 11C
→ A+B == C

continue 동작 원리 — 건너뛰기 흐름

continue는 현재 반복 회차의 나머지 코드를 건너뛰고 다음 반복으로 이동합니다.

for a in range(1,10)
↓
for b in range(1,10)
↓
if a == b ?
↓
True (같음)
continue
→ b의 다음 값으로
False (다름)
for c in range(1,10)
→ 계속 진행
↓ (False 경로)
if c==a or c==b ?
True (중복)
continue
→ c의 다음 값으로
False (유일)
조건식 검사
→ 해 발견 시 print

Step 3. 프로그램 구현 (Python)

다중 for문과 continue문을 사용하여 효율적으로 구현합니다.

Python Solution
# 3중 반복문으로 가능한 모든 (A, B, C) 조합 탐색
# 최대 9×9×9 = 729회 탐색, continue로 불필요한 연산 제거

for a in range(1, 10): # A: 1~9 (AB의 첫 자리 → 0 불가)
    for b in range(1, 10): # B: 1~9 (BA의 첫 자리 → 0 불가)
        if a == b: continue # A와 B가 같으면 다음 b로 건너뜀

        for c in range(1, 10): # C: 1~9 (CC의 첫 자리 → 0 불가)
            if c == a or c == b: continue # C가 A 또는 B와 같으면 건너뜀

            # 조건 확인: AB + BA == CC
            # → 11(a+b) == 11c → a+b == c
            if (a * 10 + b) + (b * 10 + a) == (c * 10 + c):
                print(f"A={a}, B={b}, C={c}  →  {a*10+b} + {b*10+a} = {c*10+c}")
주의사항

반복문이 중첩될수록 탐색 횟수가 기하급수적으로 늘어납니다. 변수가 4개이면 9⁴=6,561회, 8개이면 9⁸≈43,000,000회입니다. continue로 불필요한 연산을 조기에 차단하는 것이 '최적화'의 핵심입니다.

코드 실행 추적 (Trace) — a=1부터 일부 흐름

a=1로 고정했을 때, b가 변화하며 어떻게 처리되는지 추적합니다.

a b a==b? c c==a or c==b? 조건식 동작
1 1 ✅ True — — — continue (b 다음으로)
1 2 ❌ False 1 ✅ (c==a) — continue (c 다음으로)
1 2 ❌ False 2 ✅ (c==b) — continue (c 다음으로)
1 2 ❌ False 3 ❌ False 11+21=32==33? ❌ 출력 없음
1 2 ❌ False 4 ❌ False 11+21=32==44? ❌ 출력 없음
... ... ... ... ... ... ...
1 2 ❌ 3 ❌ 12+21=33==33? ✅ 🎯 출력: A=1,B=2,C=3

※ 위 Trace에서 12+21=33이 아닌 32==33임을 확인 — 실제 a=1,b=2일 때 c=3이 해입니다. (1+2=3 ✅)

Step 4. 프로그램 테스트 및 검토

🔴 원본 오류 — 예상 출력 불완전

원본에서 "예상 출력: A=1, B=2, C=3 등 여러 쌍이 나올 수 있습니다"라고만 언급했으나, 실제로는 총 36개의 해가 출력됩니다. 학생이 실행 결과와 예상이 달라 혼란을 겪을 수 있습니다. 핵심 조건은 A+B=C이며, C는 두 수의 합이므로 최소 2(=1+1 불가, 최소 1+2=3)에서 최대 9까지만 가능합니다.

실행 결과 확인

프로그램을 실행하면 조건 A+B=C를 만족하는 모든 (A,B,C) 조합이 출력됩니다.

총 해의 개수: A∈[1,9], B∈[1,9], A≠B, A+B≤9, C=A+B, C≠A, C≠B → 36쌍

출력 예시 — 일부 해 목록 (C=3~5)

A B C AB BA CC 검증
1 2 3 12 21 33 12+21=33 ✅
2 1 3 21 12 33 21+12=33 ✅
1 3 4 13 31 44 13+31=44 ✅
3 1 4 31 13 44 31+13=44 ✅
2 3 5 23 32 55 23+32=55 ✅
3 2 5 32 23 55 32+23=55 ✅
... (C=6,7,8,9인 경우도 동일 패턴으로 계속됨, 총 36쌍)
💡 패턴 발견

모든 해는 A + B = C 조건을 만족합니다. C가 고정되면 (A,B) 쌍의 수는 A+B=C이고 A≠B이고 둘 다 1~9인 쌍의 개수입니다. C=3이면 (1,2),(2,1) — 2쌍, C=9이면 (1,8),(2,7),(3,6),(4,5),(5,4),(6,3),(7,2),(8,1) — 8쌍입니다.

확장 과제 1

SEND + MORE = MONEY
8개 서로 다른 숫자를 찾는 문제입니다. 8중 for문 대신 itertools.permutations(range(10), 8)을 사용하면 훨씬 간결합니다!

확장 과제 2

사용자로부터 특정 수식을 입력받아 자동으로 알파벳을 추출하고 풀어주는 일반화된 프로그램을 고민해 보세요. set()과 itertools가 핵심입니다.

Python — itertools 활용 심화 힌트
from itertools import permutations

# SEND + MORE = MONEY: S,E,N,D,M,O,R,Y 8개 서로 다른 숫자
for perm in permutations(range(10), 8):
    S, E, N, D, M, O, R, Y = perm
    if S == 0 or M == 0: continue # 최고차항 0 불가
    SEND = S*1000 + E*100 + N*10 + D
    MORE = M*1000 + O*100 + R*10 + E
    MONEY = M*10000 + O*1000 + N*100 + E*10 + Y
    if SEND + MORE == MONEY:
        print(f"SEND={SEND}, MORE={MORE}, MONEY={MONEY}")
