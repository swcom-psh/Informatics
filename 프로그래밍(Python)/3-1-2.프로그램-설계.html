프로그램 설계: 생각의 구체화

기획서라는 '아이디어'를 '실행 가능한 설계도'로 바꾸는 시간입니다.

지난 시간에 우리는...

우리는 지난 차시에 '성적 계산기'를 만들기로 기획했습니다. 대상은 누구인지, 어떤 기능이 필요한지 정했죠. 이제 그 기획을 바탕으로 컴퓨터에게 시킬 구체적인 명령서(알고리즘)를 작성할 차례입니다!

프로그램 개발 설계서

기획 단계에서 분석한 요구 사항을 바탕으로 입출력과 처리 절차를 명확하게 정의하는 단계입니다.

1

입력 설계

어떤 데이터를 어떻게 받을까요?

  • 사용자 직접 입력: 키보드
  • 대량 데이터: Excel, CSV 파일
2

출력 설계

결과를 어디에 보여줄까요?

  • 시각적 확인: 모니터 화면
  • 데이터 저장: 결과 파일 저장

실전! 성적 계산기 설계

단순한 합산이 아닌 반영 비율(가중치)이 적용된 계산 프로세스를 설계해 봅시다.

설계 조건

  1. 입력: 중간(30%), 기말(30%), 수행(40%)의 비율과 각 점수
  2. 처리: (비율 × 점수)를 합산하는 수식 적용
  3. 출력: 최종 점수를 소수점 첫째 자리까지 표현
[입력 데이터 구성]
- 중간고사 비율 0.3, 점수 86
- 기말고사 비율 0.3, 점수 90
- 수행평가 비율 0.4, 점수 85

[출력 예상 결과]
최종 합산 점수: 86.8
주의! 제약 조건 명시

소수점 자리수 처리를 설계에서 빠뜨리면 안 됩니다! 컴퓨터는 86.8000001처럼 소수점을 길게 출력할 수 있으므로, "첫째 자리까지만 출력한다"는 규칙을 설계서에 꼭 포함해야 합니다.

알고리즘 작성

알고리즘은 문제를 해결하기 위한 구체적이고 명확한 절차를 의미합니다.

성적 산출 알고리즘

1. 각 평가 항목의 반영 비율과 점수를 입력받는다.

2. (반영 비율 × 점수)를 항목별로 계산한다.

3. 계산된 값들을 모두 더해 변수에 저장한다.

4. 소수점 첫째 자리까지 반올림하여 화면에 출력한다.

재치 있는 비유: '라면 끓이기' 알고리즘

1. 물을 냄비에 붓는다 → 2. 불을 켠다 → 3. 물이 끓는가? (No: 대기 / Yes: 면 투하) → 4. 3분 뒤 식사 시작.
조건문(물이 끓는가?)이 들어가는 순간 훌륭한 알고리즘이 됩니다!
