데이터 신뢰성 검사: 바코드 검증

수학적 알고리즘을 통해 입력된 바코드가 유효한지 판별하는 프로그램을 제작합니다.

바코드 끝자리의 비밀

우리가 마트에서 보는 바코드의 마지막 숫자는 아무렇게나 정해진 것이 아닙니다. 앞의 숫자들을 특정 공식으로 계산하여 나온 체크섬(Checksum)이죠. 이 숫자를 통해 바코드가 잘못 읽히지는 않았는지 스스로 검증할 수 있습니다.

Step 1. 프로그램 기획 (요구 사항)

8자리로 구성된 바코드가 입력되었을 때, 마지막 자리(체크 숫자)가 올바른지 확인합니다.

🔴 원본 오류 수정 — 바코드 예시 교정

원본에서는 88012345를 "유효한" 예시로 표시했으나, 실제로 알고리즘을 적용하면 체크 숫자는 5가 아니라 0이어야 합니다.
→ 올바른 유효 바코드 예시: 88012340 (체크 숫자 = 0)

Step 2. 프로그램 설계 (검증 로직)

가장 널리 쓰이는 바코드 검증 공식(단순화 버전)을 적용해 봅시다.

체크 숫자 계산 단계
  1. S1: 홀수 번째(1·3·5·7번) 자리 숫자들의 합을 구합니다.
  2. S2×3: 짝수 번째(2·4·6번) 자리 숫자들의 합에 3을 곱합니다.
  3. total = S1 + S2×3을 계산하고 remainder = total % 10으로 나머지를 구합니다.
  4. (10 - remainder) % 10이 마지막 8번째 숫자(체크 숫자)와 일치하면 유효합니다.

파이썬 인덱스(0-based) vs 자리 번호(1-based) 매핑

수학 공식은 "1번째 자리"부터 세지만, 파이썬 인덱스는 0부터 시작합니다. 코드에서 i + 1을 쓰는 이유가 바로 이것입니다.

구분 i=0 i=1 i=2 i=3 i=4 i=5 i=6 i=7
자리 번호 (i+1) 1번 (홀) 2번 (짝) 3번 (홀) 4번 (짝) 5번 (홀) 6번 (짝) 7번 (홀) 8번 (체크)
(i+1)%2==1 ? True → S1 False → S2 True → S1 False → S2 True → S1 False → S2 True → S1 check_digit
예시 88012340 8 8 0 1 2 3 4 0
(10 - remainder) % 10 — 왜 % 10이 한 번 더 필요할까요?

remainder가 0일 때 10 - 0 = 10이 되는데, 체크 숫자는 한 자리(0~9)여야 합니다. 그래서 % 10을 한 번 더 적용해 10을 0으로 만들어 줍니다.

# remainder = 5일 때: (10-5)%10 = 5%10 = 5 ✅
# remainder = 3일 때: (10-3)%10 = 7%10 = 7 ✅
# remainder = 0일 때: (10-0)%10 = 10%10 = 0 ✅ ← 이 경우를 처리!
# remainder = 0일 때 % 10 없으면: 10 → 체크 숫자로 쓸 수 없음 ❌
데이터 처리 전략
  • 문자열로 입력받은 뒤 각 자리수를 int()로 변환해야 합니다.
  • 반복문(for)과 (i+1) % 2 조건으로 홀수/짝수 자리를 구분합니다.

Step 3. 프로그램 구현 (Source Code)

Python Solution
# 1. 바코드 입력 (문자열 8자리)
barcode = input()

# 2. 앞 7자리(data)와 마지막 체크 숫자 분리
# barcode[7] → 인덱스 7 = 8번째 자리 = 체크 숫자
data = barcode[:7]
check_digit = int(barcode[7])

# 3. 홀수/짝수 자리 합계 계산
s1 = 0 # 홀수 번째(1·3·5·7번) 자리 합
s2 = 0 # 짝수 번째(2·4·6번) 자리 합

for i in range(7): # i = 0, 1, 2, 3, 4, 5, 6
    num = int(data[i])
    if (i + 1) % 2 == 1: # (i+1)이 홀수 → 홀수 번째 자리
        s1 += num
    else: # (i+1)이 짝수 → 짝수 번째 자리
        s2 += num

# 4. 체크 숫자 도출
total_sum = s1 + (s2 * 3) # 홀수합 + 짝수합×3
remainder = total_sum % 10
result = (10 - remainder) % 10 # remainder=0이면 10→0으로 처리

# 5. 결과 출력
if result == check_digit:
    print("Valid")
else:
    print("Invalid")
프로그래밍 팁: (10 - remainder) % 10

계산 결과가 10일 때 0이 되어야 하는 조건은 나머지를 한 번 더 구하는 % 10 연산으로 우아하게 해결할 수 있습니다. 조건문 없이 한 줄로 처리하는 좋은 습관입니다!

⚠️ 8자리가 아닌 입력 → IndexError

barcode[7]은 바코드가 반드시 8자리 이상일 때만 동작합니다. 7자리 이하 입력 시 IndexError: string index out of range가 발생합니다.

# 심화 과제: 길이 유효성 검사 추가
barcode = input()
if len(barcode) != 8:
    print("오류: 8자리 바코드를 입력하세요.")
else:
    # ... 이하 동일

반복 종료 후 최종 계산:

S1 = 14 + S2×3 = 12×3 = 36 = total_sum = 50
50 % 10 = 0 → (10 - 0) % 10 = 0 → result = 0 == check_digit(0) → ✅ "Valid"
