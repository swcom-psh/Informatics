고대의 지혜: 시저 암호 해독하기

아스키코드와 반복문을 활용하여 암호화된 메시지를 원래대로 되돌리는 프로그램을 설계합니다.

시저 암호란?

로마의 장군 율리우스 시저가 사용했던 암호 방식입니다. 알파벳을 일정한 거리만큼 밀어서 글자를 바꾸는 방식이죠. 예를 들어 3칸을 밀기로 했다면, 'A'는 'D'가 되고, 반대로 해독할 때는 'D'에서 3을 빼서 'A'를 찾습니다.

Step 1. 프로그램 기획 (요구 사항)

사용자가 입력한 암호문을 받아서, 정해진 규칙(왼쪽으로 3칸 이동)에 따라 해독한 결과를 출력합니다.

[해독 원리: 왼쪽으로 3칸 이동]

D
암호문
ord('D')=68
68 - 3 = 65
chr(65)='A'
A
원문

ASCII: 68 ('D') − 3 = 65 ('A')

입력 조건

알파벳 대문자로만 구성된 암호문 문자열이 입력됩니다.
예: "DEFG"

출력 조건

각 글자를 왼쪽으로 3칸 밀어 해독한 원문을 출력합니다.
예: "ABCD"

Step 2. 프로그램 설계 (핵심 함수)

컴퓨터는 문자를 숫자로 인식합니다. 이 숫자가 바로 아스키코드(ASCII)입니다.

파이썬 필수 함수
  • ord(문자): 문자를 아스키코드 정수값으로 변환합니다. 예: ord('A') = 65
  • chr(정수): 아스키코드 정수값을 다시 문자로 변환합니다. 예: chr(65) = 'A'

대문자 알파벳 ASCII 코드 참조표

A=65 ~ Z=90 사이의 값을 갖습니다. 이 범위를 벗어나면 알파벳이 아닌 다른 문자가 출력됩니다.

문자 A B C D E F G H I J K L M
ASCII 65 66 67 68 69 70 71 72 73 74 75 76 77
문자 N O P Q R S T U V W X Y Z
ASCII 78 79 80 81 82 83 84 85 86 87 88 89 90

⚠️ A(65) - 3 = 62 = '>' — 알파벳 범위(65~90)를 벗어나면 특수문자가 출력됩니다.

알파벳 순환 처리 (Advanced)

만약 'A'에서 3을 빼면 알파벳 범위를 벗어나 특수문자 '>'(ASCII 62)가 됩니다. 실제 시저 암호에서는 알파벳이 원형으로 순환하므로 A→X, B→Y, C→Z가 되어야 합니다. 이를 처리하려면 나머지 연산이 필요합니다.

# 순환 처리 공식: (ord(char) - 65 - 3) % 26 + 65
# 'A'(65): (65-65-3)%26+65 = (-3)%26+65 = 23+65 = 88 = 'X' ✅
# 'B'(66): (66-65-3)%26+65 = (-2)%26+65 = 24+65 = 89 = 'Y' ✅
# 'C'(67): (67-65-3)%26+65 = (-1)%26+65 = 25+65 = 90 = 'Z' ✅
# 'D'(68): (68-65-3)%26+65 = (0)%26+65 = 0+65 = 65 = 'A' ✅

파이썬에서 (-3) % 26 = 23임을 기억하세요. 파이썬의 나머지 연산은 항상 양수를 반환합니다.

Step 3. 프로그램 구현 (Source Code)

Python — 기본 버전 (대문자, D 이상만 안전)
# 1. 암호문 입력 받기
cipher_text = input()
plain_text = "" # 결과를 저장할 빈 문자열

# 2. 문자열의 각 문자를 하나씩 순회
for char in cipher_text:
    # ord()로 ASCII 숫자로 변환 → 3을 빼서 해독 → chr()로 다시 문자로
    code = ord(char) - 3
    plain_text += chr(code)

# 3. 최종 해독 결과 출력
print(plain_text)
왜 문자열에 +=를 쓰나요?

문자열은 한 번 만들어지면 바꿀 수 없기 때문에(불변 객체), 해독한 글자를 하나씩 새 변수에 이어 붙이는 방식을 사용합니다. 문자열이 매우 길다면 아래 리스트 방식이 더 효율적입니다.

# 성능 개선 버전: 리스트에 쌓은 뒤 join으로 합치기
result = []
for char in cipher_text:
    result.append(chr(ord(char) - 3))
print("".join(result))
✅ 완성 버전 — 알파벳 순환 처리 포함

A, B, C가 입력되어도 알파벳 범위를 벗어나지 않도록 나머지 연산을 적용한 버전입니다.

Python — 완성 버전 (순환 포함)
cipher_text = input()
plain_text = ""

for char in cipher_text:
    # (ord(char) - 65 - 3) % 26 + 65
    # 65를 빼서 0~25 범위로 정규화 → 3 빼기 → %26으로 순환 → +65로 복원
    code = (ord(char) - 65 - 3) % 26 + 65
    plain_text += chr(code)

print(plain_text)

Step 4. 프로그램 테스트 및 검증

🔴 원본 치명적 오류 — 테스트 케이스 전면 교정

오류 1 (테스트 2): 원본에서 PYTHON → MVSLRK로 표기했으나, 실제 코드 실행 결과는 MVQELK입니다.

# PYTHON 직접 계산
P=80-3=77=M,  Y=89-3=86=V,  T=84-3=81=Q,  H=72-3=69=E,  O=79-3=76=L,  N=78-3=75=K
# 정답: MVQELK (원본의 MVSLRK는 T→S, H→L, O→R 모두 틀림)

오류 2 (테스트 1): KOREA의 "정답 HLOBX"는 순환 공식 적용 시에만 맞습니다. 기본 코드로는 A(65-3=62='>')로 알파벳 범위를 이탈합니다. 두 버전을 구분해야 합니다.
