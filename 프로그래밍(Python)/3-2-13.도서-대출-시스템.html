도서 대출 시스템 개발

딕셔너리 자료구조를 활용한 도서 정보 관리 및 대출/반납 프로세스 구현

딕셔너리로 현실 시스템 만들기

실제 도서관 시스템은 어떻게 작동할까요? 도서명을 Key로, 상태를 Value로 저장하는 딕셔너리와 while 반복문을 결합하면, 사용자의 명령을 지속적으로 처리하는 실용적인 시스템을 구현할 수 있습니다. 이번 프로젝트에서는 딕셔너리 탐색, 값 수정, 조건 분기를 종합 활용합니다.

Step 1. 프로그램 기획 (문제 정의)

도서관에서 책의 상태(대출 가능, 대출 중)를 관리하고, 사용자의 명령에 따라 대출과 반납 처리를 수행하는 프로그램을 기획합니다.

핵심 개념: 딕셔너리(Dictionary)

도서명을 Key로, 도서의 상태를 Value로 매핑하여 데이터를 관리합니다.

'파이썬' : '대출가능'
'C언어' : '대출중'
'자바' : '대출가능'

Key = 도서명  |  Value = '대출가능' 또는 '대출중'

상태 전이 흐름

대출가능
초기 상태
→ 명령 1 (대출) →
← 명령 2 (반납) ←
대출중
대출 후 상태

대출 처리 (명령 1)

도서가 존재하고 '대출가능' 상태일 때만 '대출중'으로 변경합니다.

반납 처리 (명령 2)

도서가 존재하고 '대출중' 상태일 때 '대출가능'으로 변경합니다.

Step 2. 프로그램 설계 (알고리즘)

명령어 입력에 따른 조건부 로직을 설계합니다.

명령어 구조 설계

명령어 입력 형식 동작 예시
1 1 도서명 대출 실행 1 파이썬
2 2 도서명 반납 실행 2 파이썬
0 0 (도서명 불필요) 프로그램 종료 0
예외 처리

도서 목록에 없는 책 요청 시 "존재하지 않는 도서입니다." 메시지를 출력합니다.

중복 상태 처리

이미 대출된 책의 재대출, 또는 대출 가능 상태 책의 반납 요청 시 거절 메시지를 출력합니다.

핵심 문법 2가지

# 1. while True + break — 무한 반복과 탈출
while True: # 조건 없이 계속 반복
    cmd = input()
    if cmd == '0':
        break # 이 줄에서 while 루프 즉시 탈출
    # 이 아래 코드는 break 전까지 계속 실행
# 2. in 연산자 — 딕셔너리에 키가 있는지 확인
books = {"파이썬": "대출가능"}

"파이썬" in books # True (키 존재)
"수학" in books # False (키 없음)
"파이썬" not in books # False (존재하므로 not in = False)

Step 3. 프로그램 구현 (Python)

딕셔너리와 조건문을 활용한 완성 코드입니다.

🔴 원본 치명적 오류 — 반납(op==2) 분기 완전 누락

원본 코드는 Step 2에서 명령어 2 = 반납을 설계했음에도 구현 코드에 반납 처리 분기가 전혀 없습니다. 사용자가 2 파이썬을 입력해도 아무 처리가 되지 않습니다. 아래 완성 코드에서 수정되었습니다.

⚠️ cmd[1] IndexError — 도서명 없이 입력 시

op, title = cmd[0], cmd[1]에서 사용자가 1만 입력(도서명 생략)하면 IndexError: list index out of range가 발생합니다. 명령어 0은 break로 먼저 처리되므로 안전하나, 명령 1·2는 도서명이 반드시 필요합니다.

✅ 완성 버전 — 반납 분기 추가 + 주석 보강
Python Solution (완성)
# 1. 도서 데이터 초기화 (Key: 도서명, Value: 상태)
books = {"파이썬": "대출가능", "C언어": "대출가능", "자바": "대출가능"}

# 2. 사용자 명령을 반복 처리 (0 입력 시 종료)
while True:
    cmd = input("명령 입력(1:대출 2:반납 0:종료): ").split()
    if cmd[0] == '0': break # 도서명 없이 0만 입력, 즉시 종료

    op, title = cmd[0], cmd[1]

    # 3. 도서 존재 여부를 가장 먼저 확인 (이후 분기 안전 보장)
    if title not in books:
        print("존재하지 않는 도서입니다.")

    # 4. 대출 처리
    elif op == '1':
        if books[title] == "대출가능":
            books[title] = "대출중"
            print(f"{title} 대출 완료")
        else:
            print("이미 대출 중인 도서입니다.")

    # 5. ✅ [추가] 반납 처리 — 원본에 완전히 누락된 분기
    elif op == '2':
        if books[title] == "대출중":
            books[title] = "대출가능"
            print(f"{title} 반납 완료")
        else:
            print("대출 중이 아닌 도서입니다.")
학습 포인트

books[title]을 통해 데이터에 접근할 때, 해당 title(Key)이 딕셔너리에 존재하는지 가장 먼저 확인하는 것이 에러 방지의 핵심입니다. not in 검사를 바깥에 배치하면 이후 모든 분기에서 안전하게 접근할 수 있습니다.

💡 .get() 메서드 — KeyError 방지 대안

books[title]은 키가 없으면 KeyError가 발생합니다. books.get(title)은 키가 없으면 None을 반환해 더 안전합니다.

# KeyError 발생 가능
books["없는책"] # KeyError!

# 안전한 접근
books.get("없는책") # None (오류 없음)
books.get("없는책", "없음") # "없음" (기본값 지정)

코드 실행 추적 (Trace) — 5개 명령 순서대로

초기 상태: books = {"파이썬":"대출가능", "C언어":"대출가능", "자바":"대출가능"}

# 입력 op title 조건 판별 출력 books 변화
1 1 파이썬 1 파이썬 존재 ✅ / 대출가능 ✅ 파이썬 대출 완료 파이썬 → 대출중
2 1 파이썬 1 파이썬 존재 ✅ / 대출중 ❌ 이미 대출 중인 도서입니다. 변화 없음
3 1 수학 1 수학 존재 ❌ 존재하지 않는 도서입니다. 변화 없음
4 2 파이썬 2 파이썬 존재 ✅ / 대출중 ✅ 파이썬 반납 완료 파이썬 → 대출가능
5 0 — — cmd[0]=='0' → break (종료) —
