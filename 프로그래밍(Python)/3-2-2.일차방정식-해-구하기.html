일차방정식 \(ax \pm b = c\) 해 구하기

문자열 분리 기술을 활용하여 수학 문제를 해결하는 똑똑한 프로그램을 만들어 봅시다.

수학과 코딩의 만남

우리는 수학 시간에 일차방정식을 푸는 방법을 배웠습니다. 이번 프로젝트의 핵심은 사용자가 입력한 문장(문자열)에서 숫자만 쏙쏙 뽑아내어(Parsing) 컴퓨터가 계산하게 만드는 것입니다.

Step 1. 프로그램 기획 (요구 사항)

사용자가 일차방정식 형태의 식을 입력하면 컴퓨터가 \(x\)의 값을 계산해 주어야 합니다.

\(ax + b = c\)   또는   \(ax - b = c\)

입력 조건

공백을 포함한 문자열 형태로 식이 입력됩니다.
예: "3x + 5 = 20"

⚠️ 반드시 각 토큰 사이에 공백 1칸이 있어야 합니다.

출력 조건

계산된 \(x\)의 값을 출력합니다. 정수로 나누어떨어지면 정수로, 소수가 나오면 소수점 둘째 자리까지 출력하는 방식을 고려해 봅시다.

Step 2. 프로그램 설계 (해법 찾기)

일차방정식을 풀기 위해 우리가 손으로 푸는 과정을 알고리즘으로 정리해 봅시다.

해 구하기 공식 (이항 정리)

1. \(b\)를 우변으로 넘긴다: \(ax = c - b\) (혹은 \(c + b\))

2. \(a\)로 양변을 나눈다: \(x = (c \mp b) / a\)

문자열 분리(Split) 전략

입력된 "3x + 5 = 20"에서 숫자 3, 5, 20을 어떻게 분리할까요? 파이썬의 split() 함수와 문자열 슬라이싱(Slicing)을 적절히 혼합해야 합니다.

"3x + 5 = 20".split() 실행 결과

"3x"[:-1] 슬라이싱으로 계수 분리

[:-1]은 "처음부터 마지막 문자 직전까지"를 의미합니다.

'3' 'x'
"3x"[:-1] = "3"
  ⚠️
"x"[:-1]이면 어떻게 될까요?
"x"[:-1]은 빈 문자열 ""이 됩니다.
→ float("") → ValueError 발생!
→ a = 1인 경우를 별도 처리해야 합니다.

Step 3. 프로그램 구현 (Source Code)

설계한 알고리즘을 파이썬 문법으로 구현합니다. 먼저 기본 코드를 살펴보고, 이어서 예외 처리가 추가된 완성 버전을 확인해 보세요.

🔍 기본 코드 — 주석 보강 완료

원본의 주석은 parts[3]("=")을 완전히 누락해 학생이 parts[4]에 왜 바로 건너뛰는지 이해하기 어려웠습니다. 아래에서 모든 인덱스를 명시합니다.

Python — 기본 버전
# 1. 입력 받기 (예: "3x + 5 = 20")
equation = input()

# 2. 공백 기준으로 문자열 분리
parts = equation.split()
# parts[0]="3x" parts[1]="+" parts[2]="5" parts[3]="=" parts[4]="20"
# ↑ 인덱스 3의 "="은 사용하지 않으므로 parts[4]로 바로 이동합니다.

# 3. 데이터 가공 (Parsing)
a = float(parts[0][:-1]) # "3x"[:-1] → "3" → 3.0
op = parts[1]                 # "+" 또는 "-"
b = float(parts[2])            # 상수 b
c = float(parts[4])            # 우변 c (parts[3]="=" 건너뜀)

# 4. 해 계산
if op == '+':
    x = (c - b) / a # ax + b = c → x = (c - b) / a
else:
    x = (c + b) / a # ax - b = c → x = (c + b) / a

print(f"x의 값은 {x:.2f}입니다.")
✅ 완성 버전 — a=1 예외·a=0 예외 처리 포함

계수가 생략된 "x + 5 = 10" 입력 시 기본 버전은 ValueError가 발생합니다. 아래 완성 버전은 이를 해결합니다.

Python — 완성 버전 (예외 처리 포함)
equation = input()
parts = equation.split()

# ✅ a = 1 처리: "x"[:-1] = "" → float("") 오류 방지
a_str = parts[0][:-1]     # "3x" → "3", "x" → ""
a = float(a_str) if a_str else 1.0 # 빈 문자열이면 계수는 1

op = parts[1]
b = float(parts[2])
c = float(parts[4])

# ✅ a = 0 처리: 0으로 나누면 ZeroDivisionError 발생
if a == 0:
    print("오류: 계수 a가 0이면 일차방정식이 아닙니다.")
else:
    if op == '+':
        x = (c - b) / a
    else:
        x = (c + b) / a

    # ✅ 출력 형식: 정수면 정수로, 소수면 소수점 2자리로
    if x == int(x):
        print(int(x))
    else:
        print(round(x, 2))
⚠️ 출력 형식 주의

원본 코드의 f"x의 값은 {x:.2f}입니다."는 답이 5일 때 5.00으로 출력합니다. 온라인 저지(OJ)에서 정수 5를 요구하면 오답 처리될 수 있습니다.
완성 버전처럼 if x == int(x)로 정수 여부를 먼저 판별하는 것을 권장합니다.

핵심 문법: 조건부 표현식(삼항 연산자)

완성 버전의 float(a_str) if a_str else 1.0은 파이썬의 조건부 표현식입니다.

# 일반 if문으로 쓰면:
if a_str: # a_str이 빈 문자열이 아니면 True
    a = float(a_str)
else:
    a = 1.0

# 한 줄로 압축하면:
a = float(a_str) if a_str else 1.0

코드 실행 추적 (Trace) — 입력: 3x + 5 = 20

완성 버전 코드가 실제로 어떻게 실행되는지 단계별로 따라가 봅시다.

단계 코드 계산 과정 결과값
① equation = input() "3x + 5 = 20" 입력 equation = "3x + 5 = 20"
② parts = equation.split() 공백 기준 분리 ['3x', '+', '5', '=', '20']
③ a_str = parts[0][:-1] "3x"[:-1] a_str = "3"
④ a = float(a_str) if a_str else 1.0 "3" ≠ "" → float("3") a = 3.0
⑤ op, b, c 추출 parts[1], float(parts[2]), float(parts[4]) op="+", b=5.0, c=20.0
⑥ if op == '+': x = (c - b) / a (20 - 5) / 3 x = 5.0
⑦ if x == int(x): print(int(x)) 5.0 == 5 → True 출력: 5

Step 4. 테스트 및 예외 상황

프로그램이 멈추지 않도록 학생들이 자주 실수하는 부분을 점검합니다.

테스트 케이스 표 (경계값 포함)

# 입력 a op b c 기대 출력 테스트 목적
1 3x + 5 = 20 3 + 5 20 5 기본 일반 케이스
2 3x - 5 = 10 3 - 5 10 5 기본 빼기 연산자
3 x + 5 = 10 1 + 5 10 5 주의 a=1 생략 케이스
4 2x + 1 = 2 2 + 1 2 0.5 주의 소수 결과값
5 0x + 3 = 7 0 + 3 7 오류 메시지 오류 a=0 → ZeroDivisionError
6 3x+5=20 — (공백 없음) 오류 발생 오류 공백 없이 입력 시
이런 경우를 조심하세요!
  • 계수 \(a\)가 1인 경우: "x + 5 = 10" 입력 시 parts[0]은 "x"가 되고, "x"[:-1]은 빈 문자열 ""이 됩니다. float("")은 ValueError를 발생시킵니다. 완성 버전처럼 조건부 표현식으로 처리하세요.
  • 계수 \(a\)가 0인 경우: 0x + 3 = 7 같은 입력은 수학적으로 일차방정식이 아닙니다. 나눗셈 ... / a에서 ZeroDivisionError가 발생하므로 반드시 a == 0 조건 검사를 선행해야 합니다.
  • 공백 유무: split()은 공백을 기준으로 나누기 때문에, "3x+5=20"처럼 공백 없이 입력하면 분리 결과가 달라져 프로그램이 작동하지 않습니다. "각 항 사이에 공백 1칸"이라는 입력 규칙을 기획 단계에서 명확히 안내해야 합니다.
