자재 자동 라벨링: 철근 자르기

반복문과 연산자를 활용하여 자재 번호를 자동으로 생성하는 시스템을 설계합니다.

현장의 문제를 코딩으로!

공사 현장에서 긴 철근을 일정한 길이로 자른 뒤, 각각에 식별 번호(F-0001, F-0002...)를 붙여야 한다고 상상해 보세요. 손으로 쓰기엔 너무 많죠? 컴퓨터를 이용해 자재 번호를 자동으로 생성하는 프로그램을 만들어 봅시다.

Step 1. 프로그램 기획 (요구 사항)

입력된 전체 길이와 자를 길이를 바탕으로 생성 가능한 모든 자재 번호를 출력합니다.

[철근 커팅 예시 — 입력: 10 3]

3
3
3
1
← 전체 길이 10 →
F-0001
F-0002
F-0003
← 3개 생성

남은 자투리(길이 1)는 번호를 부여하지 않고 제외합니다.
count = 10 // 3 = 3  (나머지 1은 버림)

입력 조건

전체 길이 \(L\)과 자를 길이 \(k\)가 공백으로 입력됩니다.
예: 10 3

출력 조건

F-0001부터 시작하여 1씩 증가하며 번호를 한 줄에 하나씩 출력합니다.

Step 2. 프로그램 설계 (해법)

1) 몇 개를 만들 수 있을까? (산술 연산)

전체 길이를 자를 길이로 나눈 몫만큼 자재 번호를 생성할 수 있습니다.

\(count = total \,//\, cut\)

예: 10 // 3 = 3  → 3개 /  100 // 3 = 33  → 33개 /  5 // 6 = 0  → 0개(출력 없음)

2) 형식에 맞게 출력하기 (포맷팅)

단순히 1, 2, 3이 아니라 0001, 0002처럼 4자리를 맞추는 것이 포인트입니다.

  • 파이썬 f-string 활용: f"F-{i:04d}"
  • :04d는 4자리 정수로 출력하되 빈 곳은 0으로 채우라는 뜻입니다.

f"F-{i:04d}" 포맷 완전 분해

각 부분이 무엇을 의미하는지 색으로 구분하여 살펴봅니다.

F-고정 문자열
{표현식 시작
i변수(반복 번호)
:포맷 구분자
0빈 자리를 0으로 채움
4총 자릿수 4자리
d정수(decimal)
}표현식 끝
# i 값에 따른 출력 결과
i = 1     → "F-0001"
i = 10    → "F-0010"
i = 100   → "F-0100"
i = 9999 → "F-9999"
i = 10000 → "F-10000"  # ⚠️ 10000 이상이면 자릿수가 넘쳐 5자리가 됨 → 챌린지 참고!

range(1, count+1) 범위 이해하기

count = 3일 때 range(1, 4)가 만들어내는 숫자 목록입니다.

1
2
3
4

🟩 포함  |  🟥 4 제외 (끝값은 포함 안 됨)
range(시작, 끝)에서 끝값은 항상 제외됩니다. 마지막 자재 번호까지 출력하려면 반드시 count + 1이 필요합니다.

Step 3. 프로그램 구현 (Source Code)

Python Solution
# 1. 데이터 입력 (전체 길이, 자를 길이)
total, cut = map(int, input().split())

# 2. 생성 가능한 자재 개수 계산 (정수 나눗셈 — 나머지 버림)
count = total // cut

# 3. 반복문을 이용한 번호 생성 및 출력
# range(1, count+1) → 1부터 count까지 (끝값 count+1은 제외)
for i in range(1, count + 1):
    # :04d → 4자리 정수, 빈 자리는 0으로 채움
    print(f"F-{i:04d}")
반복문 range(1, count + 1) 이해하기

우리는 1번 자재부터 필요하므로 시작값을 1로 잡습니다. range()의 끝값은 포함되지 않기 때문에 count + 1까지 범위를 지정해야 마지막 번호까지 출력됩니다.

또한 count = 0이면 range(1, 1)이 되어 반복이 한 번도 실행되지 않습니다. → 출력 없음 ✅

⚠️ cut = 0 입력 시 ZeroDivisionError

total // cut에서 cut = 0이면 0으로 나누는 연산이 발생해 ZeroDivisionError: integer division or modulo by zero가 납니다.
현실적으로 "0cm로 자른다"는 입력 자체가 무의미하므로, 문제 조건상 cut ≥ 1이 보장된다면 처리 불필요합니다. 단, 방어적으로 작성할 경우 아래 코드를 참고하세요.

# 심화 과제: cut = 0 방어 처리
if cut == 0:
    print("오류: 자를 길이는 1 이상이어야 합니다.")
else:
    count = total // cut
    for i in range(1, count + 1):
        print(f"F-{i:04d}")

코드 실행 추적 (Trace) — 입력: 10 3

코드가 실제로 어떻게 실행되는지 단계별로 따라가 봅시다.

단계 코드 계산 과정 결과
① total, cut = map(int, input().split()) "10 3" → 정수 변환 total=10, cut=3
② count = total // cut 10 // 3 = 3 (나머지 1 버림) count=3
③ range(1, count+1) range(1, 4) → [1, 2, 3] 반복 3회
④ i=1, print(f"F-{1:04d}") 1 → 4자리 "0001" 출력: F-0001
⑤ i=2, print(f"F-{2:04d}") 2 → 4자리 "0002" 출력: F-0002
⑥ i=3, print(f"F-{3:04d}") 3 → 4자리 "0003" 출력: F-0003
⑦ i=4 → range 종료 4 = count+1, 조건 불만족 반복 종료

Step 4. 테스트 및 고도화

다양한 입력값으로 코드를 검증합니다. 특히 cut > total, 나누어떨어지는 경우, cut=0 경계값을 꼭 확인하세요.
