Chapter 02.

기계학습의 개념과 활용

유튜브나 넷플릭스는 어떻게 내가 좋아할 만한 영상을 쏙쏙 골라줄까요? 데이터를 통해 스스로 규칙을 깨우치는 '기계학습(Machine Learning)'의 비밀을 알아봅시다!

기계학습의 개념과 활용
기계학습이란? 기계학습의 유형 기계학습 모델 추천 시스템 원리 핵심정리 퀴즈

컴퓨터에게 가르치는 두 가지 방법

인공지능을 만드는 방법은 크게 '규칙 기반(Rule-based)'과 '학습 기반(Learning-based)'으로 나눌 수 있습니다. 기계학습(Machine Learning)은 바로 이 '학습 기반'의 대표적인 방법입니다.

전통적인 방식 (규칙 기반)

사람이 컴퓨터에게 모든 규칙을 직접 하나하나 알려주는 방식입니다.

예시:
"다리가 4개고, 꼬리가 있고, '멍멍' 소리를 내면 그것은 '강아지'라고 판단해!"
👉 단점: 예외 상황(다리가 3개인 강아지)이 생기면 컴퓨터가 고장 납니다.

기계학습 (학습 기반)

컴퓨터에게 수많은 데이터를 주고, 스스로 규칙(패턴)을 찾게 하는 방식입니다.

예시:
강아지 사진 1만 장과 고양이 사진 1만 장을 보여주며 "이게 강아지고, 저게 고양이야"라고 알려줍니다.
👉 장점: 컴퓨터가 스스로 생김새의 특징을 파악하여 새로운 사진도 잘 분류합니다!

어떻게 공부할까? 기계학습의 3가지 학습 방법

기계에게 정답을 알려주는지, 보상을 주는지 등 학습하는 방식에 따라 크게 3가지로 나눌 수 있습니다.

1. 지도학습 (Supervised Learning)

컴퓨터에게 '문제(입력 데이터)'와 문제에 대한 '정답(label)'을 함께 주면서 학습시키는 방법입니다. 선생님이 정답지를 보고 가르쳐주는 것과 같습니다.

예시: "이 사진은 고양이(정답)야"

👉 추후 배울 '예측'과 '분류'가 여기에 속해요!

2. 비지도학습 (Unsupervised Learning)

컴퓨터에게 '정답 없이 데이터만' 주고, 스스로 데이터 안에서 비슷한 특징과 패턴을 찾아내도록 하는 방법입니다.

예시: "성향이 비슷한 고객끼리 묶어봐"

👉 추후 배울 '군집화'가 여기에 속해요!

3. 강화학습 (Reinforcement Learning)

어떤 환경에서 컴퓨터가 행동을 취하면, 그 결과에 따라 '보상'이나 '벌점'을 주어 스스로 최적의 방법을 깨우치게 합니다.

예시: 강아지 훈련(간식 주기), 알파고 바둑, 다중 슬롯머신(Bandit) 및 그리드월드(GridWorld)

👉 게임 AI 에이전트, 자율주행에 많이 쓰여요!

기계학습의 유형

기계학습은 문제의 종류에 따라 값을 예측하거나, 데이터를 분류하거나, 비슷한 데이터를 군집하여 묶거나, 경험을 통해 행동을 배우는 다양한 방법을 사용합니다.

1. 지도학습: 예측 (Prediction)

"얼마일까? 몇 도일까?" 연속적인 숫자를 예측합니다.

입력 데이터

- 집의 평수, 역과의 거리 등

- 수치, 범주 등 다양한 형태 가능

출력 결과

- 주식 가격, 강수량 등

- 연속값(Real value)

대표 활용 사례

- 주식 가격 예측, 내일의 기온 예측

예측의 대표 모델 알아보기

선형 회귀 (Linear Regression)

여러 데이터 사이의 관계를 직선 형태의 규칙으로 표현하여, 새로운 데이터의 값을 예측하는 가장 기본적인 기계학습 모델입니다.

💡 예: 스마트폰 사용 시간이 늘어날수록 수면 시간이 줄어드는 경향을 직선으로 표현할 수 있습니다.

모델 실습하기
다항 회귀 (Polynomial Regression)

데이터의 관계가 직선이 아니라 곡선 형태로 나타날 때, 입력 값의 제곱이나 세제곱 같은 값을 추가하여 더 복잡한 패턴을 표현할 수 있도록 선형 회귀를 확장한 모델입니다.

💡 예: 운동 시간이 늘어날수록 체력 향상 효과가 처음에는 빠르게 증가하다가 점점 완만해지는 곡선을 표현할 수 있습니다.

모델 실습하기

2. 지도학습: 분류 (Classification)

"이건 강아지일까, 고양이일까?" 데이터를 정해진 종류(범주, 클래스)로 나눕니다.

입력 데이터

- 이메일의 제목, 내용, 보낸 사람

- 정답 레이블(Label)이 존재하는 데이터

출력 결과

- '스팸 메일' 또는 '정상 메일'

- 정해진 클래스(Class), 불연속적인 값

대표 활용 사례

- 스팸 필터링, 이미지 인식, 질병 진단

분류의 대표 모델 알아보기

로지스틱 회귀 (Logistic Regression)

이름에 '회귀'가 들어가지만 실제로는 분류에 쓰입니다. 데이터가 특정 그룹에 속할 가능성(확률)을 계산하고, 그 확률을 기준으로 새로운 데이터가 어떤 범주에 속하는지 판단하는 모델입니다.

💡 예: 이메일이 스팸일 확률을 계산하여 '스팸' 또는 '정상 메일'로 분류할 때 사용됩니다.

모델 실습하기
k-NN (최근접 이웃)

새로운 데이터가 들어왔을 때, 주변에 가장 가까이 있는 K개의 이웃 데이터들이 어느 그룹에 속하는지 보고 다수결로 자신의 그룹을 정합니다.

💡 예: 새로운 영화를 추천할 때, 나와 취향이 가장 비슷한 사람들의 평가를 참고하는 원리입니다.

모델 실습하기
의사결정 트리 (Decision Tree)

마치 스무고개 게임처럼 "날개가 있나요?", "포유류인가요?" 같은 질문(조건)을 이어가며 데이터를 분류하는 직관적인 모델입니다.

💡 예: 동물 사진을 보고 “날개가 있나요?”, “네 발로 걷나요?” 같은 질문을 통해 어떤 동물인지 분류하는 방식입니다.

모델 실습하기
SVM (서포트 벡터 머신)

서로 다른 두 그룹을 나눌 때, 두 그룹 사이의 여유 공간이 가장 넓어지도록 경계선을 찾는 분류 모델입니다.

💡 예: 게임 플레이 시간과 점수를 이용해 초보자와 숙련자를 구분하는 기준을 찾는 문제에 사용할 수 있습니다.

모델 실습하기

3. 비지도학습: 군집 (Clustering)

"정답이 없어도 끼리끼리 묶어볼까?" 비슷한 특징을 가진 데이터끼리 그룹을 만듭니다.

입력 데이터

- 쇼핑몰 고객들의 구매 기록

- 정답 레이블(Label)이 없는 데이터

출력 결과

- 비슷한 성향의 고객 그룹 (VIP, 신규 등)

- 군집(Cluster)

대표 활용 사례

- 고객 성향 세분화, 추천 시스템 기초 데이터

대표 모델 알아보기

K-평균 군집화 (K-Means Clustering)

데이터를 K개의 그룹으로 나누겠다고 미리 정해주면, 각 그룹의 중심점을 찾아가며 가장 가까운 데이터들끼리 뭉치게 하는 방법입니다.

💡 예: 비슷한 취향을 가진 학생들을 자동으로 여러 모둠으로 나눌 때 사용할 수 있습니다.

모델 실습하기
계층적 군집화 (Hierarchical Clustering)

몇 개의 그룹으로 나눌지 미리 정하지 않고, 가장 비슷한 데이터부터 차근차근 나무 가지처럼 묶어 올라가는 방식입니다.

💡 예: 생물의 진화 계통도(분류도)를 그릴 때 유사한 유전자를 가진 종끼리 묶는 원리입니다.

모델 실습하기

4. 강화학습 (Reinforcement Learning)

"시행착오를 통해 배운다!" 환경과 상호작용하며 보상(+점수)을 최대화하는 방법을 스스로 학습합니다.

입력 데이터

- 현재 처해 있는 상황/상태 (State)

- 행동에 따른 보상(Reward) 또는 벌점(Penalty)

출력 결과

- 다음 단계에서 취해야 할 행동 (Action)

- 최적의 행동 지침 (Policy)

대표 활용 사례

- 알파고(바둑), 게임 AI 에이전트, 자율주행 자동차, 로봇 제어

대표 예제 알아보기

다중 슬롯머신 (Multi-Armed Bandit)

여러 개의 슬롯머신 중 어떤 것을 당겨야 가장 많은 수익(보상)을 얻을 수 있을지 새로운 시도(탐색)와 기존의 성공(활용) 사이에서 고민하며 학습하는 기초 모델입니다.

💡 예: 웹사이트에서 A안과 B안 중 방문자 클릭률(보상)이 더 높은 디자인을 찾을 때(A/B 테스트) 사용됩니다.

예제 실습하기
그리드월드 (GridWorld)

격자 형태의 지도(환경)에서 주인공이 상하좌우로 움직이며 장애물(벌점)을 피하고 목적지(보상)에 도달하는 최적의 경로를 찾아내는 방법입니다.

💡 예: 로봇 청소기가 집안 구조를 파악하며 가장 빠르고 안전하게 청소하는 경로를 학습할 때 활용됩니다.

코드로 보는 '협업 필터링'의 기본 원리

"나와 취향이 비슷한 사람을 찾는다"는 것은 컴퓨터가 어떻게 계산할까요? 파이썬(Python) 형태의 가짜 코드(의사코드)로 그 논리를 쉽게 살펴봅시다.

recommendation_logic.py 

# 1. 사용자들의 시청 기록 데이터 수집 (어떤 채널을 구독/시청했는지)
user_me = ["게임", "음악", "요리"]
user_friend_A = ["뷰티", "패션", "브이로그"]
user_friend_B = ["게임", "음악", "IT기기 리뷰"]  # 나와 취향이 비슷함!

# 2. 취향 유사도(Similarity) 계산 함수
def check_similarity(user1, user2):
    # 겹치는 관심사의 개수를 셉니다.
    common_interests = set(user1) & set(user2)
    return len(common_interests)

# 3. 나와 가장 비슷한 사람 찾기
score_A = check_similarity(user_me, user_friend_A) # 겹치는 것 0개
score_B = check_similarity(user_me, user_friend_B) # '게임', '음악' 겹침 (2개)

if score_B > score_A:
    print("친구 B와 취향이 비슷하군요!")
    # 4. B는 봤지만 나는 안 본 영상 추천하기
    print("친구 B가 즐겨보는 'IT기기 리뷰' 영상을 추천합니다!")

"기계학습 인공지능은 처음부터 똑똑해서 다 알아맞힌다?"

아닙니다! 인공지능은 태어날 때(?) 아무것도 모르는 백지상태입니다. 기계가 학습하기 위해서는 엄청난 양의 '데이터(Data)'가 필수적입니다. 우리가 교과서로 공부하듯, 인공지능도 양질의 데이터를 많이 보며 학습해야만 정확한 결과를 낼 수 있습니다. 쓰레기 데이터를 넣으면 쓰레기 결과가 나옵니다. (Garbage In, Garbage Out)

오늘의 핵심 내용

기계학습의 개념

기계학습(Machine Learning)은 사람이 직접 규칙을 지시하지 않아도, 기계가 대량의 데이터 속에서 스스로 패턴과 규칙을 찾아내는 인공지능 기술입니다.

기계학습의 3가지 핵심 유형

예측 (Prediction)

주식 가격, 기온 등 연속적인 숫자를 예측합니다.

분류 (Classification)

스팸 메일 여부, 질병 진단 등 데이터를 정해진 범주(Class)로 나눕니다.

군집 (Clustering)
