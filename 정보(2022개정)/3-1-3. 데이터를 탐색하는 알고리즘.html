3-1-3. 데이터를 탐색하는 알고리즘

수많은 데이터 속에서 보물(원하는 값)을 찾는 효율적인 방법

💡 1억 개의 상자 안에서 물건 찾기

도서관에 무작위로 섞인 책 중 원하는 책을 찾으려면 한 권씩 다 봐야 합니다. 하지만 가나다순으로 깔끔하게 정리되어 있다면 가운데 번호를 찔러서 단번에 범위를 좁히는 것이 가능해집니다.

이처럼 프로그래밍에서 값을 콕 집어 찾아내는 기술을 탐색(Search)이라고 하며, 기초가 되는 두 논리 구조 순차 탐색과 이진 탐색을 알아봅시다.

📐 먼저 알아야 할 기초: 파이썬의 배열 번호 체계

파이썬은 0번부터 번호를 매깁니다. 일상에서 "첫 번째"라고 하면 1번이지만, 파이썬에서는 0번이 첫 번째입니다.

3[0]
7[1]
1[2]
9[3]
5[4]
data = [3, 7, 1, 9, 5]의 길이(len(data))는 5입니다.
range(5)는 0, 1, 2, 3, 4 — 즉 i가 0부터 4까지 총 5번 반복합니다.
따라서 for i in range(len(data))는 "배열의 처음(0번)부터 끝(4번)까지 한 칸씩 방문하라"는 뜻입니다.

1. 확실하지만 무식한 '순차 탐색'

순차 탐색(Sequential Search)은 데이터의 맨 앞에서부터 차례대로 단 하나씩 진짜 내 물건인지 확인하며 지워나가는 방법입니다.

🔎 목표 값: '9'를 찾아라! (배열: 1, 4, 5, 7, 9, 13, 16)

① 1단계: '1' 확인 ➔ 불일치! 다음.

② 2단계: '4' 확인 ➔ 불일치! 다음.

③ 3단계: '5' 확인 ➔ 불일치! 다음.

④ 4단계: '7' 확인 ➔ 불일치! 다음.

⑤ 5단계: '9' 확인 ➔ 찾았다! 탐색 종료. (방 다 뒤지느라 5번이나 열어봄)

  • 장점: 알고리즘이 너무 단순해서 데이터가 엉망진창이어도(정렬 안되어도) 무조건 찾아냅니다.
  • 단점: 1억 개 자료 중, 찾는 물건이 끝 쪽에 있다면 1억 번을 열어봐야 해서 너무 느립니다.
# 순차 탐색 파이썬 예제 코드
def linear_search(arr, target):
    for i in range(len(arr)):     # i는 0, 1, 2, 3, 4
        if arr[i] == target:      # 현재 위치의 값이 목표값인가?
            return i              # 맞으면 즉시 위치 반환
    return -1                     # 끝까지 못 찾으면 -1

data = [3, 7, 1, 9, 5]
result = linear_search(data, 9)
print(result)  # 출력: 3

2. 똑똑한 반반 쪼개기 '이진 탐색'

이진 탐색(Binary Search)은 우리가 흔히 아는 'Up & Down 게임'과 같습니다. 범위를 뚝딱 절반씩 잘라내며 좁혀가는 무시무시하게 빠른 탐색법입니다.

Step 1

정렬된 배열 [1, 4, 5, 7, 9, 13, 16]
가운데(7) 확인! 9는 더 크니까 왼쪽 절반은 완전히 버림.

Step 2

남은 범위 [9, 13, 16]
가운데(13) 확인! 9는 더 작으니까 오른쪽 절반을 통째로 버림.

Step 3

단 3번의 비교만으로 9를 찾아냄!
(순차 탐색은 아까 5번 걸렸음)

이진 탐색은 for i in range() 대신 while 반복을 사용합니다.
i 하나가 아니라 left, right, mid 세 개의 변수가 매 반복마다 변합니다. 
# 이진 탐색 파이썬 예제 코드
def binary_search(arr, target):
    left = 0                      # 검색 범위의 왼쪽 끝
    right = len(arr) - 1          # 검색 범위의 오른쪽 끝

    while left <= right:          # 범위가 유효한 동안 반복
        mid = (left + right) // 2 # 가운뎃점 계산

        if arr[mid] == target:    # 가운데 값이 정답이면
            return mid            # 즉시 반환
        elif arr[mid] < target:   # 가운데 값이 더 작으면
            left = mid + 1        # 오른쪽 절반으로 좁힘!
        else:                     # 가운데 값이 더 크면
            right = mid - 1       # 왼쪽 절반으로 좁힘!

data = [1, 3, 5, 7, 9, 11]  # ★ 반드시 정렬되어 있어야 함!
print(binary_search(data, 7))

3. 두 탐색의 반복 방식 비교

🐌 순차 탐색의 반복

반복 방식: for i in range(len(arr))

i 변화: 0 → 1 → 2 → 3 → … → 끝

특징: 매 반복마다 딱 한 칸 오른쪽으로 이동

비유: 복도의 방문을 1번부터 순서대로 하나씩 노크하며 열어보는 것

🚀 이진 탐색의 반복

반복 방식: while left <= right

핵심 변수: left, right, mid 세 개

특징: 매 반복마다 범위 절반을 통째로 버림

비유: 사전에서 가운데를 펼쳐보고 앞/뒤 반을 한 번에 넘기는 것

4. 탐색 효율성 승자는?

데이터 개수 📦순차 탐색 최대 비교 🐌이진 탐색 최대 비교 🚀
100개100번7번
1,000개1,000번10번
10,000개10,000번14번
1억 개1억 번최대 27번

🔑 핵심 정리 — 반복문 속 변수의 흐름

  • for i in range(len(arr))는 i를 자동으로 0, 1, 2, …, len(arr)-1까지 1씩 증가시켜 주는 구문입니다.
  • arr[i]는 "배열의 i번째 칸"을 뜻하므로, i가 바뀔 때마다 다음 칸의 값을 꺼내 확인하는 것입니다.
  • 순차 탐색은 무조건 처음부터 끝까지 한 칸씩, 이진 탐색은 가운데를 기준으로 범위를 절반씩 줄여가며 탐색합니다.
  • return을 만나면 반복 즉시 종료! 나머지 반복은 실행되지 않습니다.

▶ 정답 확인하기

• left = 0, right = 6
• mid = (0 + 6) // 2 = 3
