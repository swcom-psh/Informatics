1-1-2. 네트워크의 환경

인터넷 주소와 파일 공유는 어떻게 이루어질까?

💡 편지를 보내려면 집 주소가 필요하듯

인터넷 세상에서 내 스마트폰과 친구의 컴퓨터가 데이터를 주고받기 위해서는 각 기기를 구분할 수 있는 고유한 주소표가 있어야 합니다. 이 고유 번호를 IP 주소(Internet Protocol Address)라고 부릅니다.

1. 호수 부족 구하기 대작전! IPv4 vs IPv6

스마트폰, TV, 냉장고까지 인터넷에 붙으면서 예전 IP 주소(IPv4) 43억 개가 동이 나버렸습니다. 그래서 나온 차세대 주소가 바로 IPv6입니다.

구분 IPv4 (과거/현재) IPv6 (미래)
주소 길이 32비트 128비트
주소 표현 210.102.100.200 (10진수) fe80::fbca:5785... (16진수)
주소 갯수 약 43억 개 (2^32) 무한에 가까움 (2^128)
보안/인증 기본 제공되지 않음 기본 제공으로 강력함!

2. 외우기 힘든 숫자 주소, 문자로 바꾸기 (DNS)

우리가 네이버에 들어갈 때 `223.130.195.95` 라고 숫자를 입력하지 않고, www.naver.com 이라고 직관적인 도메인(Domain) 이름을 사용합니다.

example.com
사용자 입력
➜ DNS 서버 ➜
(전화번호부 역할)
96.7.128.198
IP로 변환하여 접속!

3. 네트워크 공유 및 클라우드

네트워크를 이용해서 우리는 프린터를 나란히 쓰고, 파일을 여러 명이 공유해서 볼 수 있습니다. (서버-클라이언트 모델)

  • 클라우드 컴퓨팅: 내 PC에 저장하지 않고, 인터넷 상의 거대한 서버에 데이터를 저장해두고 언제 어디서든 스마트폰으로 꺼내 쓰는 기술입니다. (예: 구글 드라이브)
  • NAS (Network Attached Storage): 집이나 회사 안에 직접 설치하는 '나만의 개인용 클라우드 외장하드'입니다.
  • FTP (File Transfer Protocol): 대용량 파일을 전송하기 위해 만든 특별한 고속 통신 규칙입니다.
