3-2-6. 프로그램 공유 및 성능 평가

내가 만든 코드가 완벽할까? 버그를 찾고 더 빠르게 개선하기!

🤔 에러 없이 켜진다고 전부 완벽한 무기일까?

프로그램이 단순히 돌아간다고 끝이 아닙니다. 내가 못 본 "무한 루프의 늪"이 숨어있을 수도 있고 음수 값이 들어오면 뻗어버릴 수도 있습니다.
컴퓨터 자원을 아끼고, 사용자가 어떤 값을 입력해도 버텨내는 튼튼한 코드를 위해 성능 평가와 코드 공유(코드 리뷰)는 필수입니다.

1. 프로그램 건강 검진 (체크리스트)

  • ✅
    의도한 입출력이 잘 처리되는가? 만약 사람 수가 음수(-1명)가 되는 황당한 상황이 벌어지지 않도록, 선택 구조(If문)로 "음수는 입력 불가!" 라고 막아두었는지 확인합니다.
  • ✅
    무한 반복(Infinite Loop)에 빠지지 않는가? While문의 탈출 조건이 영원히 거짓(False)이 되어, 컴퓨터가 뻗어버리는 좀비 코드가 되지 않게 점검합니다.
  • ✅
    더 효율적인 최적의 알고리즘이 있는가? 동일한 기능을 하더라도 100줄짜리 복잡한 연산보다는, 5줄짜리 빠른 연산 코드로 교체할 수 없는지 다른 친구들과 공유하며 고민합니다.

2. 실전 파일 데이터 분석 프로젝트 (Pandas 활용)

성능 좋은 프로그램을 만들면 엄청나게 많은 외부 데이터를 불러와 순식간에 분석할 수 있습니다. 데이터프레임(DataFrame)을 사용하는 pandas 라이브러리를 활용합니다.

# 1. 엑셀 파일 마법의 도구(pandas) 부르기
import pandas as pd

# 2. 엑셀이나 csv 등 파일을 불러와서 변수에 담기 (데이터프레임화)
data = pd.read_excel('nation.xlsx')

# 3. 면적이 가장 큰 나라는 어디일까?
max_nation = data[data['면적'] == data['면적'].max()]

# 4. 내림차순(높은 순서)으로 데이터 정렬하기
# ascending = False 면 높은 숫자부터 보여줍니다(내림차순).
data.sort_values(by = '유아사망률', ascending = False)

📈 데이터 시각화 보너스 지식! (matplotlib)

숫자만 보면 어지러우니까 그래프를 그릴 수 있습니다!
kind = 'bar' 는 막대그래프, 'line' 은 선그래프, 'pie' 는 원그래프, 'scatter' 는 점을 찍는 산점도 입니다.
figsize=(15, 5) 로 그래프의 사이즈를 시원하게 키울 수도 있습니다.
