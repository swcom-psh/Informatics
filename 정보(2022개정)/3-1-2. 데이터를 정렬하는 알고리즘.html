3-1-2. 데이터를 정렬하는 알고리즘

뒤죽박죽 섞인 데이터를 순서대로 멋지게 정리하는 방법!

💡 사서 선생님은 수많은 책을 어떻게 정리할까?

도서관에 있는 수만 권의 책에서 원하는 책을 빨리 찾을 수 있는 이유는, 모든 책이 일정한 기준(가나다순, 도서 번호순)에 따라 예쁘게 정렬(Sort)되어 있기 때문입니다.

컴퓨터 역시 방대한 데이터에서 값을 빠르게 찾아내기 위해 데이터를 오름차순(작은 것부터) 또는 내림차순(큰 것부터)으로 나열하는 과정이 필수적입니다. 대표적으로 가장 기초가 되는 버블 정렬과 선택 정렬에 대해 알아봅시다.

1. 거품이 위로 올라가듯 위치를 바꾸는 '버블 정렬'

버블 정렬(Bubble Sort)은 바로 옆에 있는 인접한 두 데이터의 크기를 비교해서 조건(예: 오름차순 일 때 왼쪽이 더 크다면)에 맞지 않으면 두 자리의 순서를 서로 교환(Swap)하는 방식입니다. 마치 거품이 뽀글뽀글 위로 올라가듯이 큰 값이 점차 배열의 끝으로 이동하게 됩니다.

[ 2, 5, 4, 6, 1, 3 ] 오름차순 버블 정렬 상세 (i=0, 1)

▶ i = 0 (1회차) : 맨 뒤 자리를 찾아가는 과정

j = 0 비교 : 2와 5 비교. (2 < 5) ➔ 통과 (자리 유지)

j = 1 비교 : 5와 4 비교. (5 > 4) ➔ 순서 교환! 출력: 0 번째 [2, 4, 5, 6, 1, 3]

j = 2 비교 : 5와 6 비교. (5 < 6) ➔ 통과 (자리 유지)

j = 3 비교 : 6과 1 비교. (6 > 1) ➔ 순서 교환! 출력: 0 번째 [2, 4, 5, 1, 6, 3]

j = 4 비교 : 6과 3 비교. (6 > 3) ➔ 순서 교환! 출력: 0 번째 [2, 4, 5, 1, 3, 6]

💡 i=0 종료 후, 가장 큰 값인 '6'이 맨 마지막에 확정됩니다!

▶ i = 1 (2회차)

j = 0 비교 : 2와 4 비교 ➔ 통과

j = 1 비교 : 4와 5 비교 ➔ 통과

j = 2 비교 : 5와 1 비교 ➔ 순서 교환! 출력: 1 번째 [2, 4, 1, 5, 3, 6]

j = 3 비교 : 5와 3 비교 ➔ 순서 교환! 출력: 1 번째 [2, 4, 1, 3, 5, 6]

💡 i=1 종료 후, 다음으로 큰 '5'가 뒤에서 두 번째 자리에 확정됩니다. 이렇게 전체가 맞춰질 때까지 i는 늘어나고 j 범위는 줄어들며 반복합니다.

  • 총 6개의 데이터가 있을 때, 장점: 구조가 단순하여 알고리즘을 코드로 구현하기 쉽습니다.
  • 단점: 모든 데이터를 계속 앞뒤로 짝지어 비교하고 자리도 수시로 교환해야 하므로 속도가 매우 느립니다.
# 1. 버블정렬
lst = [2, 5, 4, 6, 1, 3]
temp = 0

def bubble_sort(arr):
    n = len(arr)
    # i는 정렬을 확정지을 회차 (0부터 n-1까지)
    for i in range(n):
        # j는 앞에서부터 인접한 두 값을 짝지어 비교할 인덱스
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
              # 앞의 값이 더 크면 위치를 바꾼다 (swap)
              temp = arr[j]
              arr[j] = arr[j+1]
              arr[j+1] = temp
              # 조건문 안에서 교환될 때만 print 실행!
              print(f'{i} 번째 {arr}')
    return arr

print(f'처음: {lst}')
bubble_sort(lst)
▶ 실행 결과: 처음: [2, 5, 4, 6, 1, 3] 0 번째 [2, 4, 5, 6, 1, 3] 0 번째 [2, 4, 5, 1, 6, 3] 0 번째 [2, 4, 5, 1, 3, 6] 1 번째 [2, 4, 1, 5, 3, 6] 1 번째 [2, 4, 1, 3, 5, 6] 2 번째 [2, 1, 4, 3, 5, 6] 2 번째 [2, 1, 3, 4, 5, 6] 3 번째 [1, 2, 3, 4, 5, 6]

2. 최솟값을 쏙쏙 골라 앞으로 보내는 '선택 정렬'

선택 정렬(Selection Sort)은 전체 데이터(정렬 안 된 부분) 중에서 가장 작은(또는 큰) 값 하나를 콕 집어 찾은 뒤, 그 값을 맨 앞자리와 순서를 바꾸는 방식입니다.

[ 2, 5, 4, 6, 1, 3 ] 오름차순 선택 정렬 상세 과정

▶ i = 0 번째 반복

현재 탐색 범위 전체에서 가장 작은 값을 찾아봅니다. (j 반복)
➔ 가장 작은 값은 1 (인덱스 4)로 선택됨!

원래 i=0 위치의 값 2와 최솟값 1의 자리를 교환합니다.

➔ 출력: 0번째 [1, 5, 4, 6, 2, 3] 💡 맨 앞에 최솟값 1 확정!

▶ i = 1 번째 반복

나머지 배열(인덱스 1~5)인 [5, 4, 6, 2, 3] 안에서만 가장 작은 값을 탐색합니다.
➔ 두 번째 최솟값은 2 (인덱스 4)로 선택됨!

현재 i=1 위치의 값 5와 방금 찾은 2의 자리를 교환합니다.

➔ 출력: 1번째 [1, 2, 4, 6, 5, 3] 💡 앞에서 두 번째에 2 확정!

▶ i = 2 ~ 5 번째 반복 요약

i = 2 : 최소값 '3'(인덱스 5)을 찾아 i=2 위치('4')와 교환 ➔ 2번째 [1, 2, 3, 6, 5, 4]

i = 3 : 최소값 '4'(인덱스 5)를 찾아 i=3 위치('6')와 교환 ➔ 3번째 [1, 2, 3, 4, 5, 6]

i = 4, 5 : 이미 위치가 다 맞춰져 있어 자기 자신과 교환이 일어남 ➔ 배열 변화 없이 [1, 2, 3, 4, 5, 6] 출력 (4번째, 5번째) 

# 2. 선택정렬
lst = [2, 5, 4, 6, 1, 3]
temp = 0

print(f'처음: {lst}')
temp = 0
def selection_sort(arr):
    n = len(arr)
    # i는 현재 정렬 위치 (가장 앞자리부터 오름차순으로 채워나감)
    for i in range(n):
        min_idx = i # 일단 i번째 숫자를 최솟값의 위치라 가정
        
        # i 뒤에 남은 배열(i+1 ~ 끝)을 처음부터 끝까지 스캔해 진짜 최솟값을 찾음
        for j in range(i+1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j # 최솟값이 발견되면 갱신

        # 탐색이 끝난 후 한 회차(i)마다 딱 1번만! 
        # 처음 지정했던 i 자리의 숫자와 찾아낸 최솟값의 자리를 교환 (swap)
        temp = arr[i]
        arr[i] = arr[min_idx]
        arr[min_idx] = temp
        
        # 버블정렬과 달리 한 회차가 끝날 때마다 결과가 출력됨!
        print(f'{i}번째 {arr}')
    return arr

selection_sort(lst)
▶ 실행 결과: 처음: [2, 5, 4, 6, 1, 3] 0번째 [1, 5, 4, 6, 2, 3] 1번째 [1, 2, 4, 6, 5, 3] 2번째 [1, 2, 3, 6, 5, 4] 3번째 [1, 2, 3, 4, 5, 6] 4번째 [1, 2, 3, 4, 5, 6] 5번째 [1, 2, 3, 4, 5, 6]

⚠️ 잠깐, 엄청 헷갈리는 부분!

  • 버블 정렬과 선택 정렬은 완전히 반대 방향으로 확정됩니다.
    ➔ 버블 정렬: 한 회차가 끝나면 항상 맨 뒤쪽으로 최댓값이 확정되어 위치하게 됩니다.
    ➔ 선택 정렬: 한 회차가 끝나면 항상 맨 앞쪽으로 최솟값이 확정되어 위치하게 됩니다.
  • 교환(Swap)이 일어나는 횟수도 다릅니다.
    ➔ 버블 정렬은 옆 자리와 비교할 때마다 실시간으로 수시로 자리를 바꿉니다. 하지만 ➔ 선택 정렬은 다 돌아보며 진짜 최솟값을 하나만 달랑 찾아낸 다음 마지막에 단 1번만 자리를 교환합니다.

3. 그럼 둘 중 뭐가 더 좋을까? (정렬 알고리즘 효율성 분석)

데이터 6개( [2, 5, 4, 6, 1, 3] )를 정렬할 때 비교 횟수와 교환 횟수가 각각 어떻게 달랐는지 아래 표를 통해 정리해 봅시다.

알고리즘 동작 원리 총 비교 횟수 총 교환 횟수 특징
버블 정렬 🫧 인접한 바로 옆 데이터와 비교하여 교환 15회 8회 수시로 자리를 바꿔야 해서 교환 횟수가 많아 처리 속도가 매우 느리다.
선택 정렬 🎯 정렬되지 않은 값 중에서 최솟값을 찾아서 맨 앞자리와 교환 15회 6회 조건 비교 횟수는 버블 정렬과 완전히 같지만, 데이터 교환은 한 회차당 딱 1번만 일어난다.

4. 알고리즘의 효율성을 나타내는 '시간 복잡도' (Big-O)

⏱ 알고리즘의 실제 실행 시간 = "비교" 횟수 + "교환" 횟수

컴퓨터가 정렬을 수행할 때 가장 시간이 집중적으로 걸리는 매우 핵심적인 작업 두 가지는 두 데이터의 크기를 살펴보는 비교(Compare) 작업과, 메모리 안에서 두 데이터의 위치를 직접 뒤바꾸는 교환(Swap) 작업입니다.

따라서 어떤 정렬 알고리즘이 얼마나 효율적인지 판단하는 평가지표인 시간 복잡도(Time Complexity)는 근본적으로 총 발생한 비교 횟수와 총 발생한 교환 횟수를 더한 것에 비례하여 결정됩니다.

데이터의 총 개수를 n이라고 할 때, 처리해야 할 데이터가 점차 많아질수록 알고리즘의 연산 횟수가 어떤 비율로 증가하는지를 추세적(Trend)으로 묶어서 표현하는 방식을 점근적 표기법(대문자 O 표기법, Big-O Notation)이라고 부릅니다.

🤔 버블 정렬과 선택 정렬은 둘 다 시간 복잡도가 O(n²) 입니다. 왜 그럴까요?

  • 데이터가 6개일 때 총 비교를 몇 번 했는지 떠올려 봅시다. 5 + 4 + 3 + 2 + 1 = 총 15번 비교하게 됩니다.
  • 데이터 개수를 n개라고 두고 수학 공식으로 일반화하면 (n-1) + (n-2) + ... + 1 = n(n-1)/2 번 비교하게 됩니다.
  • 이 식에서 가장 영향력이 강한 최고차항인 n²만 기준점으로서 남겨두고 미미한 나머지(-n/2 상수항 구간 등)를 무시하여 간략히 표기한 것이 바로 O(n²)입니다. 거대한 단위의 데이터 그룹을 다룰 때 데이터 수가 10배 늘어나면 연산 시간이 100배로 제곱해서 커진다는 무시무시한 의미입니다!

✨ 핵심 결론: 이처럼 점근적 표기법 상으로는 두 알고리즘 모두 O(n²)이라는 동일하게 부진한 평가를 받지만, 위 효율성 분석 표에서 보셨듯이 수시로 값이 오가는 무겁고 느린 교환(Swap) 작업의 발생 횟수가 선택 정렬 쪽이 훨씬 적기 때문에 현실적인 체감 실행 속도에서는 선택 정렬이 버블 정렬보다 약간 더 안정적이고 빠르다는 사실! 기억해 두세요.

Q 2

[2, 6, 4, 3] 이라는 숫자를 정렬하고 있습니다. 1회차가 끝난 후에 배열 상태가 [2, 6, 4, 3] 그대로였다면, 이 알고리즘은 오름차순 버블 정렬일까요?

▶ 정답 해설 보기

정답: 아닙니다!
오름차순 버블 정렬이라면 1번(2)과 2번(6)은 안 바뀌지만, 2번(6)과 3번(4)이 바뀌어서 6이 뒤로 밀려나가야 합니다. 1회차 끝에 가장 큰 '6'이 맨 뒤에 와서 [2, 4, 3, 6]이 되어야 정상입니다. 상태가 변하지 않았다면 버블 정렬이 아닙니다.
