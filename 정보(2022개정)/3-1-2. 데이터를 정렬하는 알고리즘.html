3-1-2. 데이터를 정렬하는 알고리즘

뒤죽박죽 섞인 데이터를 순서대로 멋지게 정리하는 방법!

💡 사서 선생님은 수많은 책을 어떻게 정리할까?

도서관에 있는 수만 권의 책에서 원하는 책을 빨리 찾을 수 있는 이유는, 모든 책이 일정한 기준(가나다순, 도서 번호순)에 따라 예쁘게 정렬(Sort)되어 있기 때문입니다.

컴퓨터 역시 방대한 데이터에서 값을 빠르게 찾아내기 위해 데이터를 오름차순(작은 것부터) 또는 내림차순(큰 것부터)으로 나열하는 과정이 필수적입니다. 대표적으로 가장 기초가 되는 버블 정렬과 선택 정렬에 대해 알아봅시다.

1. 거품이 위로 올라가듯 위치를 바꾸는 '버블 정렬'

버블 정렬(Bubble Sort)은 바로 옆에 있는 **인접한 두 데이터의 크기를 비교**해서 조건(예: 오름차순 일 때 왼쪽이 더 크다면)에 맞지 않으면 두 자리의 순서를 서로 교환(Swap)하는 방식입니다. 마치 거품이 뽀글뽀글 위로 올라가듯이 큰 값이 점차 배열의 끝으로 이동하게 됩니다.

[ 데이터: 4, 8, 7, 1, 5 ] 오름차순으로 정렬하기 (1회차)

① 1번(4)과 2번(8) 비교 ➔ (4 < 8) 순서 변경 없음 ➔ 진행: [4, 8, 7, 1, 5]

② 2번(8)과 3번(7) 비교 ➔ (8 > 7) 순서 교환! 🔄 ➔ 진행: [4, 7, 8, 1, 5]

③ 3번(8)과 4번(1) 비교 ➔ (8 > 1) 순서 교환! 🔄 ➔ 진행: [4, 7, 1, 8, 5]

④ 4번(8)과 5번(5) 비교 ➔ (8 > 5) 순서 교환! 🔄 ➔ 완료: [4, 7, 1, 5, 8]

🌟 1회차가 끝났을 때 가장 큰 데이터인 '8'이 맨 마지막 위치에 확정된 것을 볼 수 있습니다. (전체 비교 4회, 교환 3회)
  • 총 5개의 데이터가 있을 때, **1회차엔 4번**, **2회차엔 3번** 등 비교해야 하는 횟수가 줄어듭니다.
  • 장점: 구조가 단순하여 알고리즘을 코드로 구현하기 무척 쉽습니다.
  • 단점: 모든 데이터를 매번 앞뒤로 비교하고 교환해야 하므로 처리 속도가 매우 느립니다.
# 버블 정렬 파이썬 코드 예제
def bubble_sort(arr):
    n = len(arr)
    for i in range(n - 1): # 전체 회차 반복
        # 맨 끝에서부터 확정된 자리를 제외하고 비교
        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]: # 앞의 데이터가 더 크다면
                # 두 데이터의 자리를 바꾼다 (교환 알고리즘)
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

data = [4, 8, 7, 1, 5]
print(bubble_sort(data)) # [1, 4, 5, 7, 8]

2. 최솟값을 쏙쏙 골라 앞으로 보내는 '선택 정렬'

선택 정렬(Selection Sort)은 전체 데이터(정렬 안 된 부분) 중에서 가장 작은(또는 큰) 값 하나를 콕 집어 찾은 뒤, 그 값을 맨 앞자리와 순서를 바꾸는 방식입니다.

[ 데이터: 4, 8, 7, 1, 5 ] О름차순으로 정렬하기 (1회차)

① 최솟값 탐색: 4, 8, 7, 1, 5 전체를 스캔해 가장 작은 숫자를 찾습니다. ➔ 찾았다! 최솟값 = 1

② 맨 앞 데이터와 위치 교환: 찾아낸 최솟값 '1'과, 현재 범위의 가장 맨 앞자리 데이터인 '4'의 자리를 바꿉니다. ➔ 순서 교환! 🔄

③ 1회차 결과: [1, 8, 7, 4, 5]

🌟 1회차가 끝났을 때 가장 작은 데이터인 '1'이 맨 앞 위치에 확정된 것을 볼 수 있습니다. (전체 비교 4회, 교환 딱 1회) 
# 선택 정렬 파이썬 코드 예제
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_index = i # 최솟값이 있다고 가정할 위치 저장
        # 정렬되지 않은 나머지 데이터에서 진짜 최솟값의 위치 탐색
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j
        # 찾아낸 최솟값을 맨 앞(i 위치)과 교환한다
        arr[i], arr[min_index] = arr[min_index], arr[i]
    return arr

data = [4, 8, 7, 1, 5]
print(selection_sort(data)) # [1, 4, 5, 7, 8]

⚠️ 잠깐, 엄청 헷갈리는 부분!

  • 버블 정렬과 선택 정렬은 완전히 반대 방향으로 확정됩니다.
    ➔ 버블 정렬: 한 회차가 끝나면 항상 맨 뒤쪽으로 최댓값이 확정되어 위치하게 됩니다.
    ➔ 선택 정렬: 한 회차가 끝나면 항상 맨 앞쪽으로 최솟값이 확정되어 위치하게 됩니다.
  • 교환(Swap)이 일어나는 횟수도 다릅니다.
    ➔ 버블 정렬은 옆 자리와 비교할 때마다 실시간으로 수시로 자리를 바꿉니다. 하지만 ➔ 선택 정렬은 다 돌아보며 진짜 **최솟값을 하나만 달랑 찾아낸 다음 마지막에 단 1번**만 자리를 교환합니다.

3. 그럼 둘 중 뭐가 더 좋을까? (정렬 알고리즘 효율성 분석)

데이터 5개를 정렬할 때 비교 횟수와 교환 횟수가 어떻게 달랐는지 살펴봅시다.

알고리즘 동작 원리 총 비교 횟수 총 교환 횟수 특징
버블 정렬 🫧 인접한 바로 옆 데이터와 비교하여 교환 10회 6회 수시로 자리를 바꿔야 해서 교환 횟수가 많아 처리 속도가 매우 느리다.
선택 정렬 🎯 정렬되지 않은 값 중에서 최솟값을 찾아서 맨 앞과 교환 10회 4회 비교하는 횟수는 버블 정렬과 같지만, 교환은 회차당 딱 1번만 일어난다.

Q 2

[2, 6, 4, 3] 이라는 숫자를 정렬하고 있습니다. 1회차가 끝난 후에 배열 상태가 [2, 6, 4, 3] 그대로였다면, 이 알고리즘은 오름차순 버블 정렬일까요?

▶ 정답 해설 보기

정답: 아닙니다!
오름차순 버블 정렬이라면 1번(2)과 2번(6)은 안 바뀌지만, 2번(6)과 3번(4)이 바뀌어서 6이 뒤로 밀려나가야 합니다. 1회차 끝에 가장 큰 '6'이 맨 뒤에 와서 [2, 4, 3, 6]이 되어야 정상입니다. 상태가 변하지 않았다면 버블 정렬이 아닙니다.
