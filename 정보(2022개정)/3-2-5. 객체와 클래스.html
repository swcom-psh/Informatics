3-2-5. 객체와 클래스

붕어빵 틀(클래스)과 붕어빵(객체)의 마법!

💡 게임 캐릭터 100마리를 한 번에 만든다면?

캐릭터마다 일일이 이름, 체력, 공격하기 코드를 따로 짠다면 너무 복잡하겠죠? 이럴 때 사용하는 것이 객체지향 프로그래밍입니다. 공통된 특징을 묶어서 "설계도"를 만들어두고, 설계도로부터 "실제 캐릭터"를 뚝딱뚝딱 찍어내는 방식입니다.

1. 커스텀 마법 상자: 사용자 정의 함수 (def)

자주 쓰는 복잡한 계산식은 나만의 함수로 만들어 두고 이름만 불러서 쓸 수 있어요.

# 비만도(BMI)를 계산해주는 마법 상자(함수) 정의
def bmi(height, weight):
    result = weight / (height ** 2)
    return round(result, 2)     # 둘째 자리까지 반올림해서 반환!

# 함수 호출 (상자 사용하기)
james_bmi = bmi(1.78, 100)
print('제임스의 BMI:', james_bmi)
매개 변수 (Input): 함수 안으로 던져주는 재료 (위 코드의 height, weight)
반환값 (Output / return): 연산 결과로 뱉어내는 값

2. 클래스(Class)와 객체(Object)

함수가 기능을 재사용한다면, 클래스는 데이터(속성)와 기능(메서드)을 하나로 묶어서 재사용하는 훨씬 발전된 형태입니다.

  • 클래스 (Class): 공통된 데이터와 기능을 묶어둔 설계도 (붕어빵 틀)
  • 객체 (Object): 클래스를 바탕으로 생성된 실체 (슈크림 붕어빵, 팥 붕어빵)
  • 인스턴스 (Instance): 객체가 컴퓨터의 메모리에 실체화되어 자리 잡았을 때 부르는 용어
# 캐릭터 설계도(클래스) 만들기
class Character:
    name = ''
    strength = 0
    
    # 공격 기능(메서드) 정의 (※ 클래스 안의 함수는 예약어 self를 꼭 붙임!)
    def attack(self, target):
        print(self.name + '가 ' + target.name + '을 공격합니다!')

# 마법사 객체 찍어내기
magician = Character()
magician.name = '마법사'

# 요정 객체 찍어내기
fairy = Character()
fairy.name = '요정'

# 메서드 (기능) 사용하기
magician.attack(fairy) # 출력: 마법사가 요정을 공격합니다!
⚠️ 에러 주의: self의 마법!
클래스 안에 메서드(작동 스킬)를 만들 때는 반드시 첫 번째 매개 변수로 self 를 적어주어야 합니다. 이것은 '이 스킬을 쓰고 있는 나 자신'을 가리킵니다!
